## 1. 스팸 필터링 문제

### 1.1 문제 정의

스팸 필터링은 이진 텍스트 분류(binary text classification) 문제이다:
- 입력: 이메일의 텍스트 내용
- 출력: 스팸(spam) 또는 정상(ham/legitimate)

### 1.2 나이브 베이즈가 스팸 필터링에 적합한 이유

1. **빠른 학습/예측 속도**: 실시간 이메일 처리에 필수적
2. **적은 메모리 사용**: 서버 자원 효율적 사용
3. **온라인 학습 가능**: 새로운 스팸 패턴에 점진적 적응
4. **양호한 분류 성능**: 텍스트 분류에서 일반적으로 우수한 성능
5. **해석 가능성**: 어떤 단어가 스팸 판단에 기여했는지 확인 가능

### 1.3 스팸 필터링의 특수한 요구사항

일반 텍스트 분류와 다른 점:
- **비대칭적 오류 비용**: FP(정상 -> 스팸)의 비용이 FN(스팸 -> 정상)보다 훨씬 크다
- **적대적 환경**: 스패머가 의도적으로 필터를 우회하려 시도한다
- **개인화 필요**: 사용자마다 스팸의 기준이 다를 수 있다
- **시간적 변화**: 스팸 패턴이 끊임없이 변화한다

## 2. 5가지 나이브 베이즈 변형

### 2.1 MultinomialNB (MNB)

McCallum & Nigam(1998)의 다항 모델:

$$P(d | c) \propto \prod_{w \in d} P(w | c)^{n_w}$$

특성: 단어의 출현 빈도(term frequency)를 사용한다.

sklearn 코드:
```python
from sklearn.naive_bayes import MultinomialNB
from sklearn.feature_extraction.text import CountVectorizer

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(documents)
model = MultinomialNB()
model.fit(X, y)
```

### 2.2 Multi-variate Bernoulli NB (MBNB)

McCallum & Nigam(1998)의 베르누이 모델:

$$P(d | c) = \prod_{w \in V} P(w | c)^{b_w} (1 - P(w | c))^{1 - b_w}$$

특성: 단어의 존재(1)/부재(0)만 사용한다.

sklearn 코드:
```python
from sklearn.naive_bayes import BernoulliNB

model = BernoulliNB()
model.fit(X, y)
```

### 2.3 MultinomialNB with Boolean Features (MNBB)

**하이브리드 접근**: 특성은 이진화(binarize)하되, 확률 모델은 다항 분포를 사용한다.

$$X_{binary} = (X > 0).astype(int)$$

이후 MultinomialNB를 적용한다.

이 접근의 **핵심 차이점**:
- BernoulliNB: 부재 단어의 확률 (1-P(w|c))을 곱한다
- MNBB: 부재 단어를 무시한다 (다항 모델의 특성)
- 동시에 MNBB는 빈도 정보를 사용하지 않는다 (이진화의 효과)

sklearn 코드:
```python
from sklearn.naive_bayes import MultinomialNB
from sklearn.feature_extraction.text import CountVectorizer

vectorizer = CountVectorizer(binary=True)
X = vectorizer.fit_transform(documents)
model = MultinomialNB()
model.fit(X, y)
```

### 2.4 MultinomialNB with TF-IDF (MNB-TFIDF)

TF-IDF 가중치를 적용한 MultinomialNB:

sklearn 코드:
```python
from sklearn.naive_bayes import MultinomialNB
from sklearn.feature_extraction.text import TfidfVectorizer

vectorizer = TfidfVectorizer()
X = vectorizer.fit_transform(documents)
model = MultinomialNB()
model.fit(X, y)
```

### 2.5 Flexible Bayes (FB)

각 단어의 조건부 분포를 **커널 밀도 추정(Kernel Density Estimation)**으로 모델링:

$$P(x_i | c) \approx \frac{1}{N_c h} \sum_{j: y_j = c} K\left(\frac{x_i - x_j^{(i)}}{h}\right)$$

가우시안 커널을 사용하여 각 특성의 분포를 비모수적으로 추정한다.

## 3. 실험 설계

### 3.1 데이터셋

### 3.2 전처리

1. HTML 태그 제거
2. 소문자 변환
3. 특수 문자 제거
4. 불용어 제거 (선택적)
5. 어간 추출(stemming) (선택적)

### 3.3 평가 지표

1. **정확도(Accuracy)**: 전체 정확도
2. **AUC(Area Under ROC Curve)**: ROC 곡선 아래 면적
3. **가중 정확도(WAcc)**: FP에 더 큰 비용을 부여한 가중 정확도

$$WAcc = \frac{\lambda \cdot TN + TP}{\lambda \cdot (TN + FP) + (TP + FN)}$$

lambda = 9 (FP가 FN보다 9배 비용이 큰 경우) 사용

## 4. 실험 결과 상세

### 4.1 AUC 기준 비교

| 데이터셋 | MNB | MBNB | MNBB | MNB-TFIDF | FB |
|---------|-----|------|------|-----------|-----|
| Ling-Spam | 0.984 | 0.974 | 0.991 | 0.987 | 0.961 |
| SpamAssassin | 0.972 | 0.951 | 0.978 | 0.974 | 0.948 |
| Enron-1 | 0.952 | 0.933 | 0.964 | 0.959 | 0.921 |
| Enron-2 | 0.943 | 0.928 | 0.958 | 0.949 | 0.917 |
| Enron-3 | 0.961 | 0.941 | 0.972 | 0.965 | 0.934 |
| Enron-6 | 0.957 | 0.933 | 0.969 | 0.959 | 0.947 |
| **평균** | **0.961** | **0.943** | **0.972** | **0.965** | **0.938** |

### 4.2 핵심 관찰

1. **MNBB가 전반적 최우수**: 모든 데이터셋에서 일관되게 높은 성능
2. **MNB-TFIDF가 2위**: TF-IDF 가중치의 효과 확인
3. **BernoulliNB가 가장 약함**: 부재 단어 모델링이 스팸 필터링에서는 오히려 해로움
4. **Flexible Bayes가 가장 낮음**: 비모수 추정의 분산이 크다

### 4.3 비용 민감 평가 (lambda = 9)

FP에 9배의 비용을 부여할 때도 MNBB가 최우수이다. 이는 MNBB가 FP를 낮게 유지하면서도 높은 전체 정확도를 달성함을 보여준다.

## 5. 왜 MNBB가 최우수인가?

### 5.1 이진화의 효과

스팸에서 "free"가 10번 등장하는 것과 3번 등장하는 것의 차이는 크지 않다. 중요한 것은 "free"가 **있느냐 없느냐**이다. 이진화는 이러한 직관을 반영한다.

### 5.2 다항 모델의 효과

BernoulliNB는 부재 단어에 대해 (1-P(w|c))를 곱한다. 어휘가 수만 개일 때, 대부분의 단어가 부재이므로 이 항이 지배적이 된다. 이는 실제로 중요한 존재 단어의 신호를 희석시킨다.

MNBB는 이진화로 빈도 정보를 제거하면서도, 부재 단어를 무시하는 다항 모델의 장점을 유지한다.

### 5.3 정보 이론적 해석

스팸 판별에서의 정보 기여:
- 핵심 스팸 단어의 존재 > 핵심 스팸 단어의 빈도 > 일반 단어의 부재
- MNBB는 첫 번째(가장 중요한)와 두 번째를 포착하면서 세 번째(노이즈)를 무시

## 6. 어휘 크기의 영향

### 6.1 소규모 어휘 (100~500 단어)

- BernoulliNB와 MNBB의 성능이 비슷
- 소수의 핵심 단어에서는 존재/부재 정보가 충분

### 6.2 대규모 어휘 (5000+ 단어)

- MNBB와 MNB가 BernoulliNB를 크게 능가
- BernoulliNB는 대규모 어휘에서 부재 단어의 곱이 지배적이 되어 성능 저하

---

## 7. 논문에서 얻는 통찰

1. **같은 나이브 베이즈라도 변형에 따라 성능이 크게 다르다**: 스팸 필터링에서 최우수(MNBB)와 최하위(FB)의 AUC 차이가 0.034로, 이는 실용적으로 유의미한 차이이다.

2. **단순한 이진화가 효과적이다**: 복잡한 TF-IDF 변환보다 단순한 이진화가 스팸 필터링에서 더 효과적일 수 있다. 이는 문제의 특성을 이해하는 것이 복잡한 기법보다 중요함을 보여준다.

3. **부재 정보의 역효과**: 대규모 어휘에서 부재 단어를 모델링하는 것은 오히려 성능을 저해할 수 있다. 이는 BernoulliNB의 근본적 한계를 보여준다.

4. **비용 민감 분류의 중요성**: 스팸 필터링에서 FP와 FN의 비대칭적 비용을 고려하는 것이 필수적이다. 단순 정확도만으로는 실제 성능을 평가하기 어렵다.

5. **실무적 권장사항**:
   - 스팸 필터링: CountVectorizer(binary=True) + MultinomialNB
   - 일반 텍스트 분류: CountVectorizer + MultinomialNB 또는 TfidfVectorizer + MultinomialNB
   - 짧은 텍스트 (SMS 등): BernoulliNB도 고려
