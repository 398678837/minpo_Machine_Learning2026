## 서지 정보

- **제목**: A Comparison of Event Models for Naive Bayes Text Classification
- **학회**: AAAI-98 Workshop, 1998

## 1. 텍스트 분류 문제 정의

### 1.1 문제 설정

텍스트 분류(text classification)는 문서 d를 미리 정의된 카테고리 집합 C = {c1, c2, ..., c_K}의 하나로 할당하는 작업이다. 베이즈 정리를 적용하면:

$$\hat{c} = \arg\max_{c \in C} P(c | d) = \arg\max_{c \in C} P(d | c) P(c)$$

- P(c): 클래스의 사전확률 (prior) - 학습 데이터에서 각 클래스의 비율로 추정
- P(d | c): 클래스 c에서 문서 d가 관측될 확률 (likelihood) - **이벤트 모델에 의해 결정**

### 1.2 이벤트 모델이란?

**이벤트 모델(event model)**은 문서의 생성 과정을 확률적으로 모델링하는 방식이다. "어떤 확률적 과정이 문서를 생성하는가?"에 대한 가정이다. 이벤트 모델의 선택에 따라 P(d | c)의 계산 방식이 완전히 달라진다.

## 2. 다변량 베르누이 모델 (Multi-variate Bernoulli Model)

### 2.1 문서의 표현

어휘 사전 V = {w1, w2, ..., w_|V|}에 대해 문서 d를 이진 벡터로 표현한다:

$$d = (b_1, b_2, ..., b_{|V|})$$

여기서 b_i는 단어 w_i가 문서에 존재하면 1, 부재하면 0이다.

### 2.2 확률 모델

$$P(d | c) = \prod_{i=1}^{|V|} P(w_i | c)^{b_i} \cdot (1 - P(w_i | c))^{1 - b_i}$$

각 단어에 대해:
- 단어가 존재(b_i = 1): P(w_i | c)의 기여
- 단어가 부재(b_i = 0): (1 - P(w_i | c))의 기여

### 2.3 파라미터 추정

라플라스 스무딩을 적용한 추정:

$$P(w_i | c) = \frac{N_{c,i} + 1}{N_c + 2}$$

- N_{c,i}: 클래스 c에서 단어 w_i를 포함하는 문서 수
- N_c: 클래스 c의 전체 문서 수

### 2.4 특징

- **단어의 존재/부재만 고려**: "free"가 1번 등장하든 10번 등장하든 동일하게 취급
- **부재 단어도 모델링**: 어떤 단어가 **없는지**도 확률에 기여 (이는 다항 모델과의 핵심 차이)
- **짧은 문서에 유리**: 단어 빈도가 대부분 0 또는 1인 짧은 문서에서 효과적

## 3. 다항 모델 (Multinomial Model)

### 3.1 문서의 표현

문서 d를 단어 빈도 벡터로 표현한다:

$$d = (n_1, n_2, ..., n_{|V|})$$

여기서 n_i는 문서에서 단어 w_i가 등장한 횟수이다. 문서의 전체 단어 수는 N_d = sum(n_i)이다.

### 3.2 확률 모델 (다항 분포)

문서를 단어들의 "가방(Bag of Words)"으로 모델링한다:

$$P(d | c) = \frac{N_d!}{\prod_{i=1}^{|V|} n_i!} \prod_{i=1}^{|V|} P(w_i | c)^{n_i}$$

다항 계수(N_d! / prod(n_i!))는 모든 클래스에 동일하므로 분류 시 무시 가능하다.

따라서 실질적인 분류 규칙은:

$$\hat{c} = \arg\max_c \left[\log P(c) + \sum_{i=1}^{|V|} n_i \log P(w_i | c)\right]$$

### 3.3 파라미터 추정

$$P(w_i | c) = \frac{T_{c,i} + 1}{\sum_{j=1}^{|V|} T_{c,j} + |V|}$$

- T_{c,i}: 클래스 c의 모든 문서에서 단어 w_i의 총 출현 횟수
- 분모: 클래스 c의 전체 단어 수 + 어휘 크기 (라플라스 스무딩)

### 3.4 특징

- **단어 빈도 활용**: "free"가 10번 등장하면 1번보다 해당 단어의 기여가 10배
- **부재 단어 무시**: n_i = 0이면 해당 단어는 로그 확률에 기여하지 않음
- **긴 문서에 유리**: 풍부한 빈도 정보를 활용할 수 있음

## 4. 두 모델의 핵심 차이

### 4.1 비교 요약

| 특성 | 베르누이 모델 | 다항 모델 |
|------|-------------|---------|
| 단어 표현 | 존재/부재 (0/1) | 빈도 (0, 1, 2, ...) |
| 부재 단어 | 확률에 기여 | 기여 없음 |
| 빈도 정보 | 무시 | 활용 |
| 문서 표현 | 이진 벡터 | 빈도 벡터 |
| 확률 분포 | 베르누이 분포 | 다항 분포 |
| sklearn 클래스 | BernoulliNB | MultinomialNB |
| 최적 상황 | 짧은 문서, 소규모 어휘 | 긴 문서, 대규모 어휘 |

### 4.2 부재 단어의 역할

베르누이 모델에서 단어 w의 부재는 (1 - P(w|c))의 확률 기여를 한다. 이는 "이 단어가 없다"는 정보가 분류에 유용할 수 있기 때문이다.

예를 들어, 스팸에서 "free"가 매우 흔하면:
- P("free" | spam) = 0.8이므로
- P("free" 부재 | spam) = 0.2
- "free"가 없는 메일은 스팸일 확률이 낮아진다

다항 모델에서는 이 정보를 사용하지 않는다.

## 6. 이론적 분석

### 6.1 왜 다항 모델이 우수한가?

1. **정보량의 차이**: 빈도 정보는 이진 정보보다 더 많은 정보를 담고 있다. 정보 이론적으로, 빈도 벡터의 엔트로피가 이진 벡터의 엔트로피보다 크다.

2. **문서 길이의 활용**: 다항 모델은 문서의 길이를 자연스럽게 반영한다. 긴 문서에서 특정 단어가 많이 등장하면, 그 신호가 증폭된다.

3. **기저율(base rate) 문제**: 베르누이 모델에서 어휘가 클 때, 대부분의 단어는 문서에 부재하므로, (1-P(w|c)) 항이 지배적이 된다. 이는 실제로 중요한 정보(존재하는 소수의 단어)의 영향을 희석시킨다.

## 7. 논문에서 얻는 통찰

1. **같은 "나이브 베이즈"라도 이벤트 모델에 따라 완전히 다른 분류기가 된다**: MultinomialNB와 BernoulliNB는 이름만 비슷할 뿐, 근본적으로 다른 확률 모델이다.

2. **텍스트 분류에서는 MultinomialNB가 기본 선택이다**: 특별한 이유가 없는 한 MultinomialNB를 먼저 시도해야 한다. BernoulliNB는 짧은 문서나 소규모 어휘에서만 고려한다.

3. **데이터 표현과 모델의 일관성**: 이진 특성(CountVectorizer + binary=True)은 BernoulliNB에, 빈도 특성(CountVectorizer)은 MultinomialNB에, TF-IDF 가중치는 MultinomialNB에 적합하다.

4. **어휘 크기의 튜닝**: MultinomialNB는 큰 어휘에서 이점이 크므로, max_features를 크게 설정하는 것이 유리하다. BernoulliNB는 소규모 어휘에서 상대적으로 양호하다.
