## 1. 서론 (Introduction) 해설

### 배경

정규화 회귀(regularized regression)는 현대 통계학과 기계학습에서 핵심적인 도구이다. 그러나 실제로 이 방법들을 사용하려면 **효율적인 계산 알고리즘**이 필수적이다. 특히 전체 정규화 경로(regularization path), 즉 $\lambda$의 넓은 범위에 대한 해를 한 번에 구해야 교차 검증 등을 통해 최적의 $\lambda$를 선택할 수 있다.

### 기존 알고리즘의 한계

1. **LARS (Least Angle Regression, Efron et al., 2004)**: Lasso의 정규화 경로를 정확히 계산할 수 있지만, Elastic Net이나 GLM으로 확장이 어렵다.
2. **내부점 방법 (Interior Point Method)**: 일반적이지만 대규모 문제에서 느리다.
3. **2차 프로그래밍 (Quadratic Programming)**: Tibshirani(1996)가 사용했으나, 변수 수가 많으면 계산량이 급증한다.

### 좌표 하강법의 재발견

좌표 하강법(coordinate descent)은 사실 오래된 알고리즘(Hildreth, 1957)이지만, Lasso 문제에 적용될 때의 효율성이 충분히 인식되지 못했다. 본 논문은 적절한 구현 기법(따뜻한 시작, 활성 집합)과 결합하면 좌표 하강법이 기존 방법들을 압도하는 성능을 보임을 입증한다.

## 2. 좌표 하강법 알고리즘 (Coordinate Descent Algorithm)

### 기본 원리

좌표 하강법의 핵심 아이디어: **한 번에 하나의 변수에 대해서만 목적 함수를 최소화하고, 이를 모든 변수에 대해 순환적으로 반복한다.**

### 가우시안 모델 (선형 회귀) 에서의 유도

Elastic Net 목적 함수:
$$f(\beta) = \frac{1}{2N}\|y - X\beta\|_2^2 + \lambda\left[\alpha\|\beta\|_1 + \frac{(1-\alpha)}{2}\|\beta\|_2^2\right]$$

$\beta_j$에 대한 편미분을 0으로 놓으면 (다른 계수는 고정):

$$\frac{\partial f}{\partial \beta_j} = -\frac{1}{N}x_j^T(y - X\beta) + \frac{1}{N}x_j^Tx_j\beta_j + \lambda(1-\alpha)\beta_j + \lambda\alpha \cdot \text{sign}(\beta_j)$$

여기서 부분 잔차(partial residual)를 정의한다:
$$r_i^{(j)} = y_i - \tilde{y}_i^{(j)} = y_i - \left(\beta_0 + \sum_{k \neq j} x_{ik}\beta_k\right)$$

그러면 $j$번째 계수의 업데이트는:

$$\beta_j \leftarrow \frac{S\left(\frac{1}{N}\sum_{i} x_{ij}r_i^{(j)},\; \alpha\lambda\right)}{\frac{1}{N}\sum_{i} x_{ij}^2 + \lambda(1-\alpha)}$$

특성이 표준화된 경우 ($\frac{1}{N}\sum_i x_{ij}^2 = 1$):

$$\beta_j \leftarrow \frac{S\left(\frac{1}{N}\sum_{i} x_{ij}r_i^{(j)},\; \alpha\lambda\right)}{1 + \lambda(1-\alpha)}$$

### 연성 임계값 연산자 (Soft Thresholding Operator)

$$S(z, \gamma) = \text{sign}(z)(|z| - \gamma)_+ = \begin{cases} z - \gamma & \text{if } z > 0 \text{ and } \gamma < |z| \\ z + \gamma & \text{if } z < 0 \text{ and } \gamma < |z| \\ 0 & \text{if } \gamma \geq |z| \end{cases}$$

이 연산자의 작동 방식:
- 입력 $z$의 절대값이 임계값 $\gamma$보다 작으면: 출력 = 0 (변수 제거)
- 입력 $z$의 절대값이 임계값 $\gamma$보다 크면: 출력 = $z$에서 $\gamma$만큼 축소

## 3. 구현 세부 사항 (Implementation Details)

### 따뜻한 시작 (Warm Starts)

전략:
1. $\lambda_{\max}$ 계산: 모든 계수가 0이 되는 최소 $\lambda$ 값
   $$\lambda_{\max} = \frac{\max_j |x_j^T y|}{N\alpha}$$
2. $\lambda_{\min} = \epsilon \cdot \lambda_{\max}$ 설정 (예: $\epsilon = 0.001$)
3. $\lambda_{\max}$에서 $\lambda_{\min}$까지 로그 스케일로 100개의 $\lambda$ 값을 생성
4. 큰 $\lambda$에서 작은 $\lambda$ 순서로 풀되, 이전 해를 초기값으로 사용

따뜻한 시작의 효과:
- 각 $\lambda$에서의 수렴이 매우 빠름 (보통 1-3회 반복)
- 전체 경로를 구하는 비용이 단일 $\lambda$에서의 비용과 거의 동일

### 활성 집합 전략 (Active Set Strategy)

관찰: Lasso 해에서 대부분의 계수는 0이다 (희소성). 0인 계수에 대한 업데이트는 불필요하다.

전략:
1. **외부 루프**: 모든 $p$개 변수에 대해 한 번 좌표 하강 수행
2. **활성 집합 식별**: 0이 아닌 계수를 가진 변수들의 집합
3. **내부 루프**: 활성 집합의 변수들에 대해서만 반복 수행
4. 내부 루프가 수렴하면 외부 루프로 돌아가서 비활성 변수 확인
5. 비활성 변수 중 진입해야 할 변수가 없으면 현재 $\lambda$에서의 해 완료

이 전략으로 계산량이 극적으로 감소한다. 특히 희소 해에서 활성 변수의 수가 $p$에 비해 매우 작을 때 효과적이다.

### KKT 조건 확인

좌표 하강법의 수렴을 KKT(Karush-Kuhn-Tucker) 조건으로 확인한다:

$$\frac{1}{N}|x_j^T(y - X\hat{\beta})| \leq \alpha\lambda \quad \text{for all } j \text{ with } \hat{\beta}_j = 0$$
$$\frac{1}{N}x_j^T(y - X\hat{\beta}) = \alpha\lambda \cdot \text{sign}(\hat{\beta}_j) + \lambda(1-\alpha)\hat{\beta}_j \quad \text{for } \hat{\beta}_j \neq 0$$

## 4. 일반화 선형 모델로의 확장 (Extension to GLMs)

### 로지스틱 회귀 (Logistic Regression)

정규화된 로지스틱 회귀의 목적 함수:

$$\min_{\beta_0, \beta} \left[ -\frac{1}{N}\sum_{i=1}^{N} \left\{ y_i(\beta_0 + x_i^T\beta) - \log(1 + e^{\beta_0 + x_i^T\beta}) \right\} + \lambda P_\alpha(\beta) \right]$$

좌표 하강법을 적용하기 위해 로그 우도를 2차 근사(quadratic approximation)한다:

$$\ell(\beta) \approx -\frac{1}{2N}\sum_{i=1}^{N} w_i(z_i - \beta_0 - x_i^T\beta)^2 + C$$

여기서:
- $w_i = p_i(1-p_i)$: 작업 가중치 (working weights)
- $z_i = \beta_0 + x_i^T\beta + \frac{y_i - p_i}{w_i}$: 작업 반응변수 (working response)
- $p_i = \frac{1}{1 + e^{-(\beta_0 + x_i^T\beta)}}$: 현재 확률 추정치

이 **반복 재가중 최소제곱법(IRLS, Iteratively Reweighted Least Squares)**과 좌표 하강법을 결합하면 정규화된 로지스틱 회귀의 해를 효율적으로 구할 수 있다.

### 포아송 회귀, 콕스 모델

유사한 2차 근사 + 좌표 하강법 전략을 포아송 회귀와 콕스 비례위험 모델에도 적용한다.

## 5. 성능 비교 (Performance Comparison)

### 벤치마크 결과

논문에서는 glmnet을 기존 방법들과 비교한다:

| 방법 | 데이터 크기 ($n \times p$) | 시간 (초) |
|------|--------------------------|-----------|
| glmnet (좌표 하강) | $1000 \times 5000$ | 0.5 |
| LARS | $1000 \times 5000$ | 3.2 |
| l1_logreg (내부점) | $1000 \times 5000$ | 68.0 |

좌표 하강법이 LARS보다 6배, 내부점 방법보다 130배 이상 빠르다. 이 차이는 데이터 크기가 커질수록 더 벌어진다.

### 확장성

- $n = 10000, p = 100000$ 규모의 문제도 몇 분 내에 전체 정규화 경로 계산 가능
- 희소 행렬(sparse matrix)을 활용하면 메모리 효율도 우수

## 8. 핵심 인사이트 (Core Insights)

1. **단순한 알고리즘의 힘**: 좌표 하강법은 개념적으로 매우 단순하지만, 적절한 구현 기법과 결합하면 최첨단 성능을 발휘한다.

2. **따뜻한 시작의 중요성**: 정규화 경로를 따라 이전 해를 재활용하는 전략이 전체 계산 효율의 핵심이다. 이는 "전체 경로를 구하는 것이 단일 해를 구하는 것만큼 쉽다"는 놀라운 결과를 낳는다.

3. **활성 집합의 효율성**: 희소 해에서는 대부분의 변수가 비활성이므로, 활성 변수만 추적하면 계산량이 극적으로 감소한다.

4. **소프트웨어의 중요성**: 아무리 좋은 이론과 알고리즘도 실용적인 소프트웨어 없이는 널리 사용되기 어렵다. glmnet의 성공은 이를 잘 보여준다.

5. **통합 프레임워크**: 가우시안, 이항, 포아송, 콕스 등 다양한 모델을 하나의 프레임워크로 통합한 것은 사용자의 학습 비용을 크게 줄인다.
