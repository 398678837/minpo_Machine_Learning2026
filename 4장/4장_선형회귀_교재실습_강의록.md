## 선형 회귀 소개 (약 8분)

### 선형 회귀란 무엇인가?

선형 회귀는 **가장 기초적인 머신러닝 모델**입니다. 이름에서 알 수 있듯이, 데이터의 관계를 **직선(선형)**으로 표현하여 예측하는 방법입니다.

여기서 핵심은 **연속형 목표 변수**를 예측한다는 점입니다. 예를 들어, 키를 예측한다면 165.5cm, 172.3cm처럼 연속적인 숫자가 나옵니다. "크다/작다"처럼 범주로 나누는 것이 아니라, 구체적인 **숫자 값**을 예측하는 것이죠.

### 왜 TOP 10에 선정되었는가?

교재에서 선형 회귀를 TOP 10 알고리즘에 포함시킨 이유는, 이것이 **기초 알고리즘**이자 **베이스라인(baseline)** 역할을 하기 때문입니다. 베이스라인이란 "기본 성능의 기준점"을 의미합니다. 어떤 복잡한 모델을 만들었을 때, "선형 회귀보다 얼마나 좋은가?"를 비교하는 출발점이 되는 것입니다.

### 장단점 정리

| 장점 | 단점 |
|------|------|
| 모델이 간단하다 | 예측력이 상대적으로 낮다 |
| 해석이 쉽다 (계수로 설명 가능) | 독립변수와 종속변수 간 선형 관계를 전제로 한다 |
| 학습 속도가 빠르다 | 이상치(outlier)에 민감하다 |

장점부터 보겠습니다. 모델이 간단하다는 것은 코드 몇 줄이면 만들 수 있다는 뜻이고, 해석이 쉽다는 것은 "이 변수가 1 증가하면 결과가 얼마나 변하는지"를 계수(coefficient)로 바로 설명할 수 있다는 뜻입니다. 이것은 실무에서 매우 중요합니다. 경영진에게 "왜 이런 예측이 나왔나요?"라고 질문받을 때 명확하게 답할 수 있습니다.

단점은 예측력이 상대적으로 낮다는 점입니다. 복잡한 비선형 관계를 포착하지 못합니다. 그리고 독립변수와 종속변수 사이에 선형 관계가 있다는 것을 전제로 합니다. 실제 데이터가 곡선 관계를 가지면 성능이 떨어집니다.

### 어디에 유용한가?

선형 회귀는 **연속형 변수를 예측**하는 곳이라면 어디든 사용할 수 있습니다.

- **BMI(체질량지수)** 예측
- **매출액** 예측
- **전력 사용량** 예측
- **보험료** 예측 (오늘 우리가 할 것!)

이런 곳에서 특히 유용합니다. 오늘은 보험료 예측을 직접 해보겠습니다.

이 구분은 나중에 더 복잡한 모델을 다룰 때 매우 중요해집니다.

이제 각 통계량을 설명하겠습니다.

| 통계량 | 의미 | 예시 (age) |
|--------|------|-----------|

## 4.4 전처리: 학습셋과 시험셋 나누기 (약 10분)

### 4.4.1 변수와 데이터셋을 나누는 이유

데이터를 확인했으니, 이제 **전처리(Preprocessing)** 단계입니다. 오늘 할 전처리는 데이터를 나누는 것입니다.

두 가지를 나눠야 합니다.

**첫째, 독립변수와 종속변수를 나눕니다.**

- **독립변수(Independent Variable, Feature)**: 예측에 사용하는 변수들입니다. age, sex, bmi, children, smoker가 독립변수입니다.
- **종속변수(Dependent Variable, Target)**: 예측하고 싶은 변수입니다. charges가 종속변수입니다.

왜 나누는가? 모델에게 "이 정보(독립변수)를 가지고 이것(종속변수)을 예측해라"라고 알려줘야 하기 때문입니다. 통상적으로 독립변수는 **X**, 종속변수는 **y**로 표기합니다.

**둘째, 학습셋(Train Set)과 시험셋(Test Set)을 나눕니다.**

이것은 매우 중요한 개념입니다. 비유를 들어보겠습니다.

시험을 준비한다고 합시다. 문제집으로 공부(학습)를 하고, 실제 시험(테스트)을 봅니다. 만약 시험 문제가 문제집과 똑같다면? 100점을 받겠지만, 그건 실력이 아닙니다. 진짜 실력을 평가하려면 **처음 보는 문제**로 시험을 봐야 합니다.

머신러닝도 마찬가지입니다. 모델이 학습에 사용한 데이터로 평가하면 성능이 좋게 나오지만, 그건 실제 성능이 아닙니다. 이것을 **오버피팅(Overfitting, 과적합)**이라고 합니다. 학습 데이터에만 지나치게 맞춰져서 새로운 데이터에는 성능이 떨어지는 현상입니다.

이를 방지하기 위해 데이터를 **학습셋**과 **시험셋**으로 나눕니다.

- **학습셋(Train Set)**: 모델이 학습(공부)하는 데이터
- **시험셋(Test Set)**: 모델의 성능을 평가(시험)하는 데이터

비율은 보통 **7:3** 또는 **8:2**를 사용합니다. 오늘은 8:2로 나누겠습니다.

### 4.4.2 데이터셋 나누기

자, 코드를 작성합시다. 새 셀에 다음을 입력하세요.

```python
X = data[['age', 'sex', 'bmi', 'children', 'smoker']]  # 독립변수
y = data['charges']  # 종속변수
```

첫 줄에서 `data[['age', 'sex', 'bmi', 'children', 'smoker']]`는 data에서 이 5개 컬럼만 선택하는 것입니다. 대괄호가 두 개인 이유는, 여러 컬럼을 동시에 선택할 때는 리스트 형태로 넘겨야 하기 때문입니다. 결과를 X에 저장합니다.

두 번째 줄에서 `data['charges']`는 charges 컬럼 하나만 선택하는 것입니다. 대괄호가 하나입니다. 결과를 y에 저장합니다.

이제 학습셋과 시험셋으로 나눕니다. 새 셀에 다음을 입력하세요.

```python
from sklearn.model_selection import train_test_split  # 사이킷런 임포트
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2, random_state=100)  # 데이터셋 분할
```

한 줄씩 설명하겠습니다.

`from sklearn.model_selection import train_test_split`: 사이킷런(scikit-learn) 라이브러리에서 `train_test_split` 함수를 가져옵니다. 이 함수가 데이터를 자동으로 나눠줍니다.

`train_test_split(X, y, test_size=0.2, random_state=100)`: 핵심 매개변수를 보겠습니다.

- **X, y**: 나눌 데이터입니다. 독립변수와 종속변수를 함께 넣어줍니다.
- **test_size=0.2**: 시험셋의 비율입니다. 0.2는 20%를 의미합니다. 즉, 80%는 학습용, 20%는 시험용입니다.
- **random_state=100**: 랜덤 시드를 고정합니다. 데이터를 나눌 때 무작위로 섞는데, 이 시드를 고정하면 **항상 같은 결과**가 나옵니다. 재현성(reproducibility)을 위해 중요합니다. 여러분이 제가 보여주는 것과 같은 결과를 얻으려면 반드시 random_state=100을 사용해야 합니다.

결과로 **4개의 데이터셋**이 만들어집니다.

| 데이터셋 | 내용 | 크기 |
|----------|------|------|
| X_train | 학습용 독립변수 | 1070행 x 5열 |
| X_test | 시험용 독립변수 | 268행 x 5열 |
| y_train | 학습용 종속변수 | 1070개 |
| y_test | 시험용 종속변수 | 268개 |

1338개 데이터 중 80%인 1070개가 학습용, 20%인 268개가 시험용으로 나뉩니다.

## 4.7 예측 모델 평가하기 (약 18분)

모델이 예측을 했으니, 이제 "얼마나 잘 예측했는가?"를 평가해야 합니다. 교재에서는 세 가지 방법을 제시합니다.

### 4.7.1 테이블로 평가하기

가장 직관적인 방법입니다. 실제값과 예측값을 나란히 놓고 비교합니다. 새 셀에 다음을 입력하세요.

```python
comparison = pd.DataFrame({'actual': y_test, 'pred': pred})
comparison
```

`pd.DataFrame()`으로 새로운 표를 만듭니다. 'actual' 열에 실제값(y_test), 'pred' 열에 예측값(pred)을 넣습니다.

실행하면 **268 rows x 2 columns** 테이블이 나옵니다. 268개의 시험 데이터에 대해 실제값과 예측값이 나란히 보입니다.

몇 개만 살펴보세요. actual과 pred가 비슷한 행도 있고, 많이 차이나는 행도 있을 것입니다. 그런데 268개를 일일이 비교하기는 어렵겠죠? 그래서 그래프와 통계적 방법이 필요합니다.

### 4.7.2 그래프로 평가하기

산점도(Scatter Plot)를 그려봅시다. 새 셀에 다음을 입력하세요.

```python

import seaborn as sns
plt.figure(figsize=(10,10))
sns.scatterplot(x='actual', y='pred', data=comparison)
```

`matplotlib`과 `seaborn`은 파이썬의 시각화 라이브러리입니다. `scatterplot`은 산점도를 그리는 함수입니다. x축에 actual(실제값), y축에 pred(예측값)을 넣습니다.

실행하면 산점도가 나타납니다.

이 그래프를 어떻게 해석하는지 설명하겠습니다.

만약 모든 예측이 완벽하다면, 즉 예측값 = 실제값이라면, 모든 점이 **대각선(45도 직선)** 위에 놓일 것입니다. 이 대각선을 상상해 보세요. 왼쪽 아래에서 오른쪽 위로 가는 직선입니다.

이 대각선을 기준으로 세 영역을 구분할 수 있습니다.

- **영역 1 (대각선 위 근처)**: 정확한 예측. 실제값과 예측값이 비슷합니다.
- **영역 2 (대각선 위쪽)**: 과대예측(Overestimation). 실제보다 높게 예측했습니다.
- **영역 3 (대각선 아래쪽)**: 과소예측(Underestimation). 실제보다 낮게 예측했습니다.

산점도를 보면, 대각선 근처에 몰려있는 점들도 있지만, 상당히 벗어나 있는 점들도 있을 것입니다. 특히 실제 보험료가 높은 쪽에서 과소예측이 많이 보일 것입니다. 이것은 선형 회귀 모델의 한계를 보여줍니다.

### 4.7.3 통계적인 방법으로 평가하기: RMSE

그래프는 직관적이지만, "정확히 얼마나 좋은가?"를 숫자 하나로 표현하기 어렵습니다. 그래서 통계적 지표가 필요합니다.

**오차(Error)의 문제**

오차 = 예측값 - 실제값입니다. 예를 들어,
- 실제값 100, 예측값 120이면 오차 = +20
- 실제값 100, 예측값 80이면 오차 = -20

이 두 오차를 더하면 0이 됩니다. 양수와 음수가 상쇄되기 때문입니다. 이것은 문제가 있습니다. 실제로는 두 번 다 20만큼 틀렸는데, 합계는 0이 나오니까요. 그래서 오차의 부호를 없애는 방법이 필요합니다.

**두 가지 해결책:**

1. **절대값을 씌운다** → MAE (Mean Absolute Error, 평균 절대 오차)
2. **제곱을 한다** → MSE (Mean Squared Error, 평균 제곱 오차)

교재에서는 Table A와 Table B 예시를 통해 MAE와 MSE의 차이를 설명합니다.

- **Table A**: 오차가 고르게 분포 (1, 2, 1, 2)
- **Table B**: 오차가 한 곳에 집중 (0, 0, 0, 6)

MAE는 두 테이블 모두 1.5로 동일합니다. 하지만 MSE는 Table B가 더 큽니다. 왜냐하면 MSE는 큰 오차에 **제곱으로 패널티**를 주기 때문입니다. 6이라는 큰 오차는 제곱하면 36이 됩니다.

따라서 MSE는 **큰 오차에 민감**합니다. 이런 특성 때문에 많은 경우에 MSE가 선호됩니다.

**주요 평가 지표 정리:**

| 지표 | 이름 | 수식 개념 | 특징 |
|------|------|-----------|------|
| MAE | 평균 절대 오차 | 오차의 절대값 평균 | 직관적 |
| MSE | 평균 제곱 오차 | 오차의 제곱 평균 | 큰 오차에 민감 |
| RMSE | 평균 제곱근 오차 | MSE에 루트 | 가장 일반적, 단위가 원래 단위와 동일 |
| R^2 | 결정 계수 | SSR / SST | 0~1 사이, 1에 가까울수록 좋음 |

**RMSE**는 MSE에 루트를 씌운 것입니다. MSE는 제곱 때문에 단위가 원래 단위의 제곱이 됩니다 (예: 달러^2). 루트를 씌우면 원래 단위(달러)로 돌아옵니다. 그래서 RMSE가 **가장 일반적으로 사용**됩니다.

**R^2 (결정 계수)**는 독립변수가 종속변수를 **얼마나 잘 설명하는지의 비율**입니다. 0에서 1 사이의 값을 가지며, **1에 가까울수록 좋습니다**. 예를 들어 R^2 = 0.74라면, 독립변수가 종속변수의 변동을 74% 설명한다는 뜻입니다.

자, 이제 코드로 계산해봅시다. 새 셀에 다음을 입력하세요.

```python
from sklearn.metrics import mean_squared_error
mean_squared_error(y_test, pred) ** 0.5  # RMSE 계산
```

`mean_squared_error(y_test, pred)`는 MSE를 계산합니다. 여기에 `** 0.5`(0.5 제곱 = 루트)를 붙이면 RMSE가 됩니다.

결과: 약 **5684.93**

이 값은 "평균적으로 약 5685 정도의 오차가 있다"는 뜻입니다. 보험료 단위(달러)로 약 5685달러 정도 틀린다는 것이죠.

같은 결과를 더 간단하게 얻을 수도 있습니다.

```python
mean_squared_error(y_test, pred, squared=False)  # 같은 결과
```

`squared=False`를 넣으면 MSE 대신 RMSE를 바로 계산해줍니다. 같은 약 5684.93이 나올 것입니다.

이제 R^2 값도 확인합시다.

```python
model.score(X_train, y_train)  # R² 계산
```

결과: 약 **0.7368**

R^2 = 0.7368은 독립변수(age, sex, bmi, children, smoker)가 종속변수(charges)의 변동을 **약 73.7% 설명**한다는 뜻입니다. 나머지 26.3%는 이 모델로 설명하지 못하는 부분입니다. 나쁘지 않지만, 아직 개선의 여지가 있다는 뜻이기도 합니다.

## 4.8 이해하기: 선형 회귀 (약 12분)

지금까지 실습을 통해 "사용법"을 배웠습니다. 이제 **원리**를 이해해봅시다.

### 선형 회귀의 수학적 원리

중학교 때 배운 일차 함수를 기억하시나요?

> **y = ax + b**

여기서 a는 **기울기(slope)**, b는 **y절편(intercept)**입니다.

선형 회귀가 하는 일은 바로 이것입니다. 데이터에 가장 잘 맞는 **직선**을 찾는 것입니다. 독립변수가 여러 개면 다음과 같이 확장됩니다.

> y = a1*x1 + a2*x2 + a3*x3 + ... + b

우리 문제에서는:

> charges = a1*age + a2*sex + a3*bmi + a4*children + a5*smoker + b

모델이 학습한다는 것은 이 수식에서 **a1, a2, a3, a4, a5, b 값을 찾는 것**입니다.

### 손실 함수(Loss Function)

그런데 "가장 잘 맞는 직선"이란 무엇일까요? 이것을 수학적으로 정의해야 합니다.

데이터 점들과 직선 사이의 **거리(오차)의 합**이 가장 작은 직선이 "가장 잘 맞는 직선"입니다. 구체적으로는, 오차의 제곱의 합을 최소화합니다. 이것을 **손실 함수(Loss Function)** 또는 **비용 함수(Cost Function)**라고 합니다.

선형 회귀에서는 이 최소화 문제를 수학적으로 정확하게 풀 수 있습니다 (최소제곱법, Ordinary Least Squares). 그래서 학습 속도가 빠른 것입니다.

### SST, SSR, SSE

R^2를 이해하기 위해 세 가지 개념을 알아야 합니다.

- **SST (Total Sum of Squares)**: 전체 변동. "데이터가 평균으로부터 얼마나 흩어져 있는가?"
- **SSR (Regression Sum of Squares)**: 회귀로 설명되는 변동. "모델이 설명하는 부분"
- **SSE (Error Sum of Squares)**: 오차 변동. "모델이 설명하지 못하는 부분"

관계식: **SST = SSR + SSE**

그리고 **R^2 = SSR / SST** 입니다.

R^2가 1이면 SSR = SST, 즉 모델이 모든 변동을 설명한다는 뜻입니다. R^2가 0이면 모델이 아무것도 설명하지 못한다는 뜻입니다.

### 계수(Coefficient) 확인

이제 모델이 학습한 계수(기울기)와 y절편을 확인해봅시다. 새 셀에 다음을 입력하세요.

```python
model.coef_  # 계수 확인
```

이것은 배열(array)로 나와서 어떤 변수의 계수인지 알기 어렵습니다. 보기 좋게 정리합시다.

```python
pd.Series(model.coef_, index=X.columns)
```

결과:

```
age        264.799803
sex         17.344661
bmi        297.514806
children   469.339602
smoker   23469.280173
```

y절편도 확인합시다.

```python
model.intercept_  # y절편
```

결과: **-11576.999976112367**

### 완성된 수식

이제 모델이 학습한 완전한 수식을 쓸 수 있습니다.

> **charges = 264.8 x age + 17.3 x sex + 297.5 x bmi + 469.3 x children + 23469.3 x smoker - 11577.0**

### 계수의 해석

이 수식의 각 계수가 의미하는 바를 해석해봅시다. 이것이 선형 회귀의 가장 큰 장점입니다.

- **age의 계수 264.8**: 다른 변수가 동일할 때, 나이가 1살 증가하면 보험료가 약 **265달러 증가**합니다.
- **sex의 계수 17.3**: 다른 변수가 동일할 때, 남성(1)이 여성(0)보다 보험료가 약 **17달러 높습니다**. (차이가 매우 작습니다)
- **bmi의 계수 297.5**: 다른 변수가 동일할 때, BMI가 1 증가하면 보험료가 약 **298달러 증가**합니다.
- **children의 계수 469.3**: 다른 변수가 동일할 때, 자녀가 1명 증가하면 보험료가 약 **469달러 증가**합니다.
- **smoker의 계수 23469.3**: 다른 변수가 동일할 때, 흡연자(1)는 비흡연자(0)보다 보험료가 약 **23,469달러 높습니다**!

보세요, **smoker의 계수가 압도적으로 큽니다.** 흡연 여부가 보험료에 가장 큰 영향을 미치는 변수라는 것을 알 수 있습니다. 이것은 직관적으로도 이해가 됩니다. 흡연은 건강에 큰 영향을 미치고, 따라서 보험사에서 흡연자에게 더 높은 보험료를 청구하는 것입니다.

반면, sex의 계수는 17.3으로 매우 작습니다. 성별은 보험료에 거의 영향을 미치지 않는다는 뜻입니다.

이런 식으로 각 변수의 영향력을 해석할 수 있는 것이 선형 회귀의 큰 장점입니다.

### 전체 코드 요약

마지막으로, 오늘 작성한 코드를 한눈에 정리하겠습니다.

```python
# 1. 라이브러리 및 데이터 불러오기
import pandas as pd
file_url = 'https://media.githubusercontent.com/media/musthave-ML10/data_source/main/insurance.csv'
data = pd.read_csv(file_url)

# 2. 데이터 확인
data.head()
data.info()
round(data.describe(), 2)

# 3. 전처리 (데이터 분할)
X = data[['age', 'sex', 'bmi', 'children', 'smoker']]
y = data['charges']
from sklearn.model_selection import train_test_split
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2, random_state=100)

# 4. 모델링
from sklearn.linear_model import LinearRegression
model = LinearRegression()
model.fit(X_train, y_train)

# 5. 예측
pred = model.predict(X_test)

# 6. 평가
comparison = pd.DataFrame({'actual': y_test, 'pred': pred})
import matplotlib.pyplot as plt
import seaborn as sns
plt.figure(figsize=(10,10))
sns.scatterplot(x='actual', y='pred', data=comparison)

from sklearn.metrics import mean_squared_error
print('RMSE:', mean_squared_error(y_test, pred, squared=False))
print('R²:', model.score(X_train, y_train))

# 7. 모델 해석
pd.Series(model.coef_, index=X.columns)
print('절편:', model.intercept_)
```
