## 1. 서론: KNN의 장점과 한계

### 1.1 KNN의 장점

KNN은 다음과 같은 장점으로 인해 널리 사용되는 분류 알고리즘이다:
1. **단순성**: 알고리즘이 직관적이고 구현이 쉽다
2. **비모수적**: 데이터의 분포에 대한 가정이 필요 없다
3. **다중 클래스 지원**: 자연스럽게 여러 클래스를 처리한다
4. **비선형 결정 경계**: 복잡한 결정 경계를 표현할 수 있다

### 1.2 KNN의 한계

1. **높은 예측 비용**: O(Nd) - N개의 학습 데이터, d차원
2. **높은 저장 비용**: 전체 학습 데이터를 메모리에 보관
3. **노이즈 민감성**: K=1일 때 특히 노이즈에 취약
4. **불필요한 특성의 영향**: 관련 없는 특성이 거리를 왜곡
5. **클래스 불균형 문제**: 다수 클래스에 편향된 예측

## 2. K값 선택 전략

### 2.1 교차 검증 기반 K 선택

가장 기본적이면서도 효과적인 방법은 **K-Fold 교차 검증**이다:

```
알고리즘: 교차 검증 기반 K 선택
입력: 학습 데이터 D, K의 후보 범위 [K_min, K_max]
출력: 최적 K*

for K = K_min to K_max:
    학습 데이터를 F개의 폴드로 분할
    for each fold f:
        f를 제외한 데이터로 KNN(K) 학습
        f에 대해 예측 및 정확도 계산
    CV_score(K) = 평균 정확도
return K* = argmax_K CV_score(K)
```

### 2.2 경험적 가이드라인

- **sqrt(N) 규칙**: K = sqrt(N)을 시작점으로 사용 (N은 학습 데이터 수)
- **홀수 선택**: 이진 분류에서 동률을 방지하기 위해 홀수 K를 선호
- **K 범위**: 일반적으로 1 <= K <= 30 범위에서 탐색

## 3. 거리 메트릭 비교

### 3.1 민코프스키 거리 패밀리

$$d_p(x, y) = \left(\sum_{k=1}^{d} |x_k - y_k|^p\right)^{1/p}$$

| p 값 | 거리 이름 | 특성 |
|------|----------|------|
| p = 1 | 맨해튼 (L1) | 이상치에 강건, 고차원에서 양호 |
| p = 2 | 유클리드 (L2) | 가장 널리 사용, 직관적 |
| p -> inf | 체비셰프 (L-inf) | 최대 차이에 기반 |

### 3.2 실험 결과

논문의 실험에서는 데이터셋에 따라 최적의 거리 메트릭이 다르지만, 전반적으로 유클리드 거리(L2)가 안정적인 성능을 보인다. 고차원 데이터에서는 맨해튼 거리(L1)가 유클리드 거리보다 우수한 경우가 있다 (Beyer et al., 1999와 일치).

## 4. 가중 KNN (Weighted KNN)

### 4.1 거리 가중 투표 (Distance-Weighted Voting)

균일 가중치 대신 거리에 반비례하는 가중치를 사용한다:

$$\hat{y} = \arg\max_c \sum_{i \in N_K(x)} w_i \cdot \mathbb{1}(y_i = c)$$

여기서 가중치 w_i는:

$$w_i = \frac{1}{d(x, x_i)^2}$$

또는:

$$w_i = \frac{d(x, x_K) - d(x, x_i)}{d(x, x_K) - d(x, x_1)}$$

여기서 x_K는 K번째 이웃, x_1은 최근접 이웃이다.

### 4.2 가중 KNN의 장점

- 가까운 이웃의 영향력이 크므로, 더 세밀한 결정 경계를 형성
- K가 다소 클 때도 가까운 이웃이 결과를 주도하므로, K 선택에 덜 민감
- 같은 K에서 균일 가중치보다 일반적으로 우수한 성능

---

## 5. 특성 가중 KNN (Feature-Weighted KNN)

### 5.1 특성 관련성 평가

각 특성의 분류 관련성을 평가하여 가중치를 부여한다.

**정보 이득(Information Gain) 기반 가중치**:

$$w_k = IG(k) = H(Y) - H(Y | X_k)$$

여기서 H는 엔트로피, Y는 클래스 레이블, X_k는 k번째 특성이다.

**상관 계수 기반 가중치**:

$$w_k = |\text{corr}(X_k, Y)|$$

### 5.2 가중 거리 함수

$$d_w(x, y) = \sqrt{\sum_{k=1}^{d} w_k (x_k - y_k)^2}$$

w_k가 큰 특성은 거리 계산에 더 큰 영향을 미치고, w_k가 작은 특성은 영향이 줄어든다. 극단적으로 w_k = 0이면 해당 특성은 완전히 무시된다 (특성 선택과 동일한 효과).

## 6. KNN 모델 기반 접근법

### 6.1 핵심 아이디어

전통적 KNN(게으른 학습)은 모든 학습 데이터를 저장하고 예측 시 전체 데이터를 탐색한다. **모델 기반 접근법**은 학습 단계에서 불필요한 데이터를 제거하여 예측 효율을 향상시킨다.

### 6.2 인스턴스 편집 (Instance Editing)

**편집된 최근접 이웃(Edited Nearest Neighbor, ENN)** 규칙:

```
알고리즘: ENN
1. 각 학습 데이터 x_i에 대해 K개의 최근접 이웃을 찾는다
2. x_i의 클래스가 K개 이웃의 다수결과 다르면, x_i를 제거한다
3. 남은 데이터만 저장하여 예측에 사용한다
```

ENN은 **결정 경계 근처의 노이즈 데이터를 제거**하여 분류 성능을 향상시킨다.

### 6.3 축소된 최근접 이웃 (Condensed Nearest Neighbor, CNN)

```
알고리즘: CNN
1. 저장 집합 S를 빈 집합으로 초기화
2. 각 클래스에서 하나의 데이터를 S에 추가
3. 반복:
   각 학습 데이터 x_i에 대해:
     S를 사용하여 1-NN 분류를 수행
     분류가 틀리면 x_i를 S에 추가
   더 이상 추가할 데이터가 없으면 종료
4. S만 저장하여 예측에 사용
```

CNN은 **결정 경계 근처의 핵심 데이터만 보존**하여 저장 공간과 예측 시간을 절약한다.

## 7. 노이즈 감소 전략

### 7.1 노이즈 식별

노이즈 데이터 포인트의 특성:
- 자신의 K개 이웃 중 대다수가 다른 클래스에 속함
- 결정 경계 내부가 아닌, 다른 클래스의 영역에 위치함

### 7.2 K값에 따른 노이즈 감소 효과

- K가 작을 때 (K=1): 노이즈에 매우 민감, 하나의 잘못된 이웃이 결과를 좌우
- K가 커질 때: 다수결에 의해 노이즈의 영향이 감소
- 최적 K: 노이즈 감소와 결정 경계의 정확도 사이의 균형

---

## 8. 논문에서 얻는 통찰

1. **KNN은 하나의 알고리즘이 아니라 프레임워크이다**: K 선택, 거리 메트릭, 가중치 방법, 투표 방법 등 다양한 구성 요소를 조합하여 성능을 최적화할 수 있다.

2. **특성 가중의 중요성**: 모든 특성이 동일하게 중요하지 않다. 특성 가중치를 적절히 설정하면 불필요한 특성의 영향을 줄이고 분류 성능을 향상시킬 수 있다.

3. **데이터 편집의 유효성**: 모든 학습 데이터가 동일하게 유용하지 않다. 노이즈 데이터를 제거하고 핵심 데이터만 보존하면 성능과 효율성을 모두 향상시킬 수 있다.

4. **실무 가이드라인**:
   - 먼저 교차 검증으로 최적 K를 찾는다
   - 특성 스케일링을 반드시 수행한다
   - 특성 가중 또는 특성 선택을 고려한다
   - 노이즈가 많은 데이터에서는 ENN 등의 편집 기법을 적용한다
   - 가중 KNN을 기본으로 사용한다

5. **다른 알고리즘과의 앙상블 가능성**: KNN은 다른 분류기와의 앙상블에서도 유용하다. 서로 다른 K, 거리 메트릭, 특성 부분집합을 사용하는 여러 KNN 분류기를 앙상블할 수 있다.
