## 개요 (300+ words)

본 논문은 Pandas 라이브러리의 창시자인 Wes McKinney가 Pandas의 설계 동기, 핵심 데이터 구조, 기능을 최초로 학술적으로 발표한 논문이다. SciPy 2010 컨퍼런스에서 발표된 이 논문은 파이썬 데이터 과학 생태계의 역사에서 가장 영향력 있는 논문 중 하나로 평가된다.

McKinney는 금융 데이터 분석가로 일하면서, 파이썬에 R의 data.frame이나 MATLAB의 구조체(structure)에 해당하는 고수준 데이터 구조가 부재하다는 것을 절감했다. NumPy의 ndarray는 동질적(homogeneous) 수치 데이터에는 탁월하지만, 실제 통계 분석에서 다루는 이질적(heterogeneous) 표 형식 데이터(문자열, 숫자, 날짜가 혼합된 데이터)를 다루기에는 부족했다. 이 간극을 메우기 위해 Pandas를 개발하게 되었다.

논문에서 소개하는 Pandas의 두 가지 핵심 데이터 구조는 Series와 DataFrame이다. Series는 인덱스가 부여된 1차원 배열로, NumPy 배열에 라벨(label)을 추가한 것이다. DataFrame은 각 열이 서로 다른 데이터 타입을 가질 수 있는 2차원 표 구조로, SQL 테이블이나 엑셀 스프레드시트와 유사하다.

McKinney는 Pandas의 핵심 설계 원칙을 명확히 제시한다. 첫째, 인덱스 기반의 자동 데이터 정렬(automatic data alignment)이다. 서로 다른 인덱스를 가진 데이터를 연산할 때, Pandas가 자동으로 인덱스를 맞춰준다. 둘째, 결측치(missing data)의 체계적 처리이다. NaN을 사용한 결측치 표현과 isna(), fillna(), dropna() 등의 처리 함수를 기본 제공한다. 셋째, groupby 연산을 통한 분할-적용-결합(split-apply-combine) 패턴의 지원이다. 넷째, merge/join을 통한 관계형 데이터 결합이다.

또한 시계열(time series) 데이터 처리에 대한 강력한 지원도 소개된다. 날짜/시간 인덱스, 리샘플링(resampling), 이동 윈도우(rolling window) 연산 등은 금융 데이터 분석에서의 McKinney의 경험이 반영된 것이다.

이 논문은 발표 당시에는 작은 프로젝트의 소개에 불과했지만, 이후 Pandas가 파이썬 데이터 과학의 핵심 라이브러리로 성장하면서 역사적 의의를 갖게 되었다.

## 핵심 기여 (Key Contributions)

1. **DataFrame 데이터 구조 도입**: R의 data.frame에 대응하는 파이썬 네이티브 데이터 구조
2. **자동 데이터 정렬**: 인덱스 기반의 자동 정렬로 데이터 조작의 안전성과 편의성 확보
3. **결측치 처리 체계화**: NaN 기반의 일관된 결측치 표현 및 처리 메커니즘
4. **Split-Apply-Combine 패턴**: groupby()를 통한 그룹별 집계의 직관적 구현
5. **관계형 데이터 결합**: merge/join을 통한 SQL 스타일의 데이터 결합 지원

## 기계학습 과목과의 관련성

Pandas의 DataFrame은 ML 워크플로우에서 데이터를 로딩하고, 탐색하고, 전처리하는 핵심 도구이다. 3장에서 학습하는 DataFrame 인덱싱, groupby, merge, 결측치 처리 등의 기능이 모두 이 논문에서 소개된 개념들이다. 이후 4~13장에서 다양한 ML 알고리즘을 적용할 때, 데이터는 항상 DataFrame 형태로 로딩되고 전처리된 후 NumPy 배열로 변환되어 모델에 입력된다.
