## 1. 서론: 데이터 정리의 중요성

### 1.1 현실적 비용
데이터 분석가의 업무 시간 중 **50~80%**가 데이터 정리(cleaning, tidying, wrangling)에 소비된다. 이는 실제 분석, 모델링, 시각화보다 훨씬 많은 시간이다.

### 1.2 이론의 부재
데이터 정리는 오랫동안 "지루하지만 필요한 작업"으로 여겨졌을 뿐, 체계적인 이론이 없었다. Wickham은 이 간극을 메우기 위해 "깔끔한 데이터"의 원칙을 제안한다.

---

## 2. 깔끔한 데이터의 세 가지 원칙

### 원칙 1: 각 변수는 하나의 열을 구성한다
```
깔끔함:                          지저분함:
| 이름   | 과목  | 점수 |        | 이름   | 수학 | 영어 |
|--------|-------|------|        |--------|------|------|
| 홍길동 | 수학  | 90   |        | 홍길동 | 90   | 85   |
| 홍길동 | 영어  | 85   |        | 김철수 | 78   | 92   |
| 김철수 | 수학  | 78   |
| 김철수 | 영어  | 92   |
```

왼쪽의 "long format"이 깔끔한 형식이다. '과목'과 '점수'가 각각 하나의 열이다.

### 원칙 2: 각 관측은 하나의 행을 구성한다
하나의 관측(한 학생의 한 과목 점수)이 하나의 행에 대응한다.

### 원칙 3: 각 관측 단위의 유형은 하나의 테이블을 구성한다
학생 정보(이름, 나이)와 성적 정보(과목, 점수)는 별도의 테이블로 분리하고, 키(학생 ID)로 연결한다.

---

## 3. 지저분한 데이터의 다섯 가지 유형

### 유형 1: 열 헤더가 변수명이 아닌 값

```python
# 지저분한 형태
messy = pd.DataFrame({
    '이름': ['홍길동', '김철수'],
    '2023': [85, 92],     # 열 헤더가 연도 '값'
    '2024': [90, 88]
})

# 깔끔한 형태로 변환 (melt)
tidy = messy.melt(
    id_vars='이름',
    var_name='연도',
    value_name='점수'
)
#      이름   연도  점수
# 0  홍길동  2023    85
# 1  김철수  2023    92
# 2  홍길동  2024    90
# 3  김철수  2024    88
```

### 유형 2: 하나의 열에 여러 변수 저장

```python
# 지저분한 형태
messy = pd.DataFrame({
    '항목': ['수학_중간', '수학_기말', '영어_중간', '영어_기말'],
    '점수': [85, 90, 78, 88]
})

# 깔끔한 형태로 변환 (문자열 분리)
tidy = messy['항목'].str.split('_', expand=True)
tidy.columns = ['과목', '시험']
tidy['점수'] = messy['점수']
#   과목  시험  점수
# 0  수학  중간   85
# 1  수학  기말   90
# 2  영어  중간   78
# 3  영어  기말   88
```

### 유형 3: 변수가 행과 열에 걸쳐 저장

데이터가 행 방향과 열 방향 모두에 변수를 포함하는 경우. melt와 pivot의 조합으로 해결한다.

### 유형 4: 하나의 테이블에 여러 관측 단위 혼재

학생의 인적 정보와 성적 정보가 하나의 테이블에 중복 저장되는 경우. 정규화(normalization)를 통해 별도의 테이블로 분리해야 한다.

### 유형 5: 하나의 관측 단위가 여러 테이블에 분산

월별로 별도 파일에 저장된 데이터 등. concat을 통해 하나의 테이블로 통합한다.

```python
# 여러 파일의 데이터를 하나로 통합
import glob

files = glob.glob('data/sales_*.csv')
dfs = [pd.read_csv(f) for f in files]
combined = pd.concat(dfs, ignore_index=True)
```

## 4. Pandas에서의 깔끔한 데이터 도구

### 4.1 Wide -> Long 변환 (melt)

```python
# Wide format (지저분할 수 있음)
wide = pd.DataFrame({
    '이름': ['홍길동', '김철수'],
    '수학': [90, 78],
    '영어': [85, 92],
    '과학': [88, 80]
})

# Long format으로 변환 (깔끔한 형태)
long = wide.melt(
    id_vars='이름',
    var_name='과목',
    value_name='점수'
)
```

### 4.2 Long -> Wide 변환 (pivot)

```python
# Long format을 다시 Wide format으로
wide_again = long.pivot(
    index='이름',
    columns='과목',
    values='점수'
)
```

### 4.3 stack/unstack

```python
# 멀티인덱스 활용
stacked = wide.set_index('이름').stack()
stacked.index.names = ['이름', '과목']
```

### 4.4 pivot_table (집계 포함)

```python
# 그룹별 집계와 피벗을 동시에
sales.pivot_table(
    values='매출',
    index='월',
    columns='카테고리',
    aggfunc='sum',
    fill_value=0
)
```

## 5. 깔끔한 데이터의 장점

### 5.1 분석 코드의 재사용성
```python
# 깔끔한 데이터이면, 동일한 코드로 다양한 분석 가능
# 어떤 '과목'이든 동일한 코드로 분석
tidy_df.groupby('과목')['점수'].agg(['mean', 'std', 'count'])
```

### 5.2 시각화의 단순화
```python
import seaborn as sns

# 깔끔한 데이터 = seaborn에 바로 전달 가능
sns.boxplot(data=tidy_df, x='과목', y='점수')
sns.barplot(data=tidy_df, x='과목', y='점수', hue='학기')
```

### 5.3 ML 파이프라인과의 호환

scikit-learn이 요구하는 입력 형식:
```

y: (n_samples,) = 깔끔한 데이터에서 타깃 열을 선택
```

## 6. Wide vs Long: 언제 무엇을 사용할까

| 관점 | Wide Format | Long Format |
|------|-------------|-------------|
| 인간의 읽기 | 직관적 | 행이 많아 읽기 어려울 수 있음 |
| 분석/시각화 도구 | 일부 함수에 유리 | 대부분의 도구가 선호 |
| ML 입력 | 특성 행렬로 직접 사용 가능 | 피벗 후 사용 |
| 저장 효율 | 희소한 경우 비효율적 | 결측치 없이 저장 가능 |
| 새 범주 추가 | 열 추가 필요 | 행 추가로 충분 |

---

## 7. 핵심 인사이트

1. **데이터 정리는 분석의 80%**: 이 단계를 체계화하면 전체 생산성이 크게 향상된다.
2. **형식이 분석을 결정한다**: 같은 데이터라도 형식에 따라 분석의 용이성이 크게 달라진다.
3. **표준화의 가치**: 깔끔한 형식은 도구와 코드의 표준화를 가능하게 하여 재사용성을 높인다.
4. **melt/pivot은 핵심 기술**: Wide <-> Long 변환은 데이터 전처리의 가장 기본적인 연산이다.
5. **관계형 사고의 중요성**: 관측 단위별로 테이블을 분리하고 키로 연결하는 정규화 사고가 필요하다.

## 기계학습 관점에서의 의의

ML의 입력 데이터는 "행=샘플, 열=특성"의 형식이어야 한다. 이것은 깔끔한 데이터의 원칙과 정확히 일치한다. 실제 ML 프로젝트에서 데이터가 처음부터 이 형식인 경우는 드물며, 대부분 여러 소스에서 수집된 지저분한 데이터를 깔끔한 형태로 변환하는 과정이 필요하다. 3장에서 학습하는 Pandas의 데이터 조작 기능(melt, pivot, merge, groupby 등)은 바로 이 변환 과정을 수행하는 도구이다. 본 논문의 원칙을 내재화하면, 데이터 전처리를 체계적으로 수행할 수 있다.
