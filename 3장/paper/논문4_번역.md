## 1. Pandas 개요

Pandas는 파이썬에서 **구조화된 데이터(structured data)**를 다루기 위한 핵심 라이브러리이다. 2008년 Wes McKinney가 금융 회사(AQR Capital Management)에서 시계열 데이터 분석의 필요에 의해 개발을 시작했으며, 현재는 데이터 과학의 사실상 표준 도구이다.

### 핵심 철학
1. **실용성 우선**: 실무 데이터 분석가의 요구에 맞춘 설계
2. **성능과 편의성의 균형**: NumPy 기반의 성능 + 직관적 API
3. **포괄적 기능**: 데이터 입출력부터 분석까지 원스톱 제공

## 2. 핵심 데이터 구조 상세

### 2.1 Series

```python
import pandas as pd

# Series = 인덱스 + 값
s = pd.Series([10, 20, 30, 40], index=['a', 'b', 'c', 'd'], name='점수')

# 벡터화 연산
s * 2        # 각 원소에 2를 곱함
s > 20       # 불리언 Series 반환
s[s > 20]    # 불리언 인덱싱 (20 초과인 값만)
```

### 2.2 DataFrame

```python
# DataFrame = 여러 Series의 집합
df = pd.DataFrame({
    '이름': ['홍길동', '김철수', '이영희', '박민수'],
    '나이': [25, 30, 28, 35],
    '점수': [85.5, 92.3, 88.1, 76.8],
    '학과': ['컴공', '통계', '컴공', '수학']
})

# 기본 정보 확인
df.info()       # 타입, 결측치 정보
df.describe()   # 수치형 열의 통계량
df.shape        # (4, 4) = 4행 4열
```

## 3. 데이터 입출력 (I/O)

| 형식 | 읽기 | 쓰기 |
|------|------|------|
| CSV | `pd.read_csv()` | `df.to_csv()` |
| Excel | `pd.read_excel()` | `df.to_excel()` |
| SQL | `pd.read_sql()` | `df.to_sql()` |
| JSON | `pd.read_json()` | `df.to_json()` |
| Parquet | `pd.read_parquet()` | `df.to_parquet()` |
| HTML | `pd.read_html()` | `df.to_html()` |
| Clipboard | `pd.read_clipboard()` | `df.to_clipboard()` |

```python
# CSV 파일 읽기
df = pd.read_csv('data.csv', encoding='utf-8', index_col=0)

# SQL 데이터베이스에서 읽기
import sqlite3
conn = sqlite3.connect('database.db')
df = pd.read_sql('SELECT * FROM table_name', conn)

# Parquet 형식으로 저장 (대용량 데이터에 효율적)
df.to_parquet('data.parquet')
```

## 4. 데이터 선택과 인덱싱

### 4.1 열 선택

```python
# 단일 열 -> Series
df['이름']

# 복수 열 -> DataFrame
df[['이름', '점수']]
```

### 4.2 행 선택

```python
# loc: 라벨 기반
df.loc[0]           # 인덱스 0인 행
df.loc[0:2]         # 인덱스 0~2 (끝 포함)
df.loc[0:2, '이름':'점수']  # 행과 열 동시 선택

# iloc: 위치 기반
df.iloc[0]          # 첫 번째 행
df.iloc[0:2]        # 첫 두 행 (끝 미포함)
df.iloc[0:2, 0:3]   # 행과 열 동시 선택
```

### 4.3 조건 기반 필터링

```python
# 불리언 인덱싱
df[df['점수'] > 80]                     # 점수 80 초과
df[(df['학과'] == '컴공') & (df['나이'] < 30)]  # 복합 조건
df.query('점수 > 80 and 학과 == "컴공"')        # query 메서드
```

## 5. 데이터 정제

### 5.1 결측치 처리

```python
# 결측치 확인
df.isna().sum()        # 열별 결측치 수
df.isna().sum().sum()  # 전체 결측치 수

# 결측치 제거
df.dropna()            # 결측치 있는 행 제거
df.dropna(axis=1)      # 결측치 있는 열 제거
df.dropna(thresh=3)    # 비결측치가 3개 미만인 행 제거

# 결측치 대체
df.fillna(0)                    # 0으로 대체
df.fillna(df.mean())            # 열 평균으로 대체
df.fillna(method='ffill')       # 앞의 값으로 채우기
df.interpolate()                # 보간법
```

### 5.2 중복 처리

```python
df.duplicated()        # 중복 행 확인
df.drop_duplicates()   # 중복 행 제거
```

### 5.3 타입 변환

```python
df['나이'] = df['나이'].astype(float)
df['날짜'] = pd.to_datetime(df['날짜'])
df['학과'] = df['학과'].astype('category')
```

## 6. 그룹별 연산 (GroupBy)

### 6.1 기본 집계

```python
# 단일 집계
df.groupby('학과')['점수'].mean()

# 다중 집계
df.groupby('학과')['점수'].agg(['mean', 'std', 'count'])

# Named Aggregation
df.groupby('학과').agg(
    평균점수=('점수', 'mean'),
    최고점수=('점수', 'max'),
    학생수=('이름', 'count')
)
```

### 6.2 변환 (Transform)

그룹별 통계량을 원래 DataFrame의 각 행에 매핑:

```python
# 그룹별 표준화 (z-score)
df['점수_표준화'] = df.groupby('학과')['점수'].transform(
    lambda x: (x - x.mean()) / x.std()
)
```

### 6.3 필터 (Filter)

조건을 만족하는 그룹만 선택:

```python
# 평균 점수가 80 이상인 학과만 선택
df.groupby('학과').filter(lambda x: x['점수'].mean() >= 80)
```

## 7. 데이터 결합

### 7.1 merge (SQL 스타일)

```python
# Inner Join (기본)
pd.merge(df1, df2, on='key')

# Left Join
pd.merge(df1, df2, on='key', how='left')

# 다중 키
pd.merge(df1, df2, on=['key1', 'key2'])
```

### 7.2 concat (연결)

```python
# 행 방향 연결
pd.concat([df1, df2], axis=0, ignore_index=True)

# 열 방향 연결
pd.concat([df1, df2], axis=1)
```

## 8. Method Chaining 패턴

Pandas의 현대적 사용 방식은 메서드 체이닝이다:

```python
result = (
    df
    .query('나이 >= 25')
    .assign(등급=lambda x: pd.cut(x['점수'], bins=[0, 70, 85, 100], labels=['C', 'B', 'A']))
    .groupby('등급')
    .agg(평균점수=('점수', 'mean'), 인원=('이름', 'count'))
    .sort_values('평균점수', ascending=False)
)
```

## 9. 핵심 인사이트

1. **DataFrame은 사고의 도구**: 데이터를 표로 구조화하면 분석이 명확해진다
2. **일관된 API**: 대부분의 연산이 유사한 패턴(선택 -> 연산 -> 결과)을 따른다
3. **결측치 처리가 핵심**: 실제 데이터는 항상 불완전하며, 이를 다루는 능력이 데이터 분석의 기본
4. **GroupBy는 가장 강력한 도구**: split-apply-combine 패턴으로 복잡한 분석을 간결하게 표현
5. **Method chaining으로 가독성 확보**: 여러 단계의 변환을 연결하여 데이터 파이프라인 구성

## 기계학습 관점에서의 의의

Pandas는 ML 프로젝트의 시작점이다. `pd.read_csv()`로 데이터를 로드하고, `info()`, `describe()`로 탐색하고, 결측치를 처리하고, 피처를 생성하고, `values` 또는 `to_numpy()`로 NumPy 배열로 변환하여 모델에 입력하는 것이 표준적인 ML 워크플로우이다. 3장에서 학습하는 모든 Pandas 기능은 이 워크플로우의 핵심 구성 요소이다.
