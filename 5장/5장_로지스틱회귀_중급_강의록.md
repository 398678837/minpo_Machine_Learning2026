## 1. 수리적 기초: 로지스틱 회귀의 유도

### 1.1 일반화 선형 모형(GLM)에서의 위치

로지스틱 회귀는 **일반화 선형 모형(Generalized Linear Model, GLM)** 의 한 특수 사례이다. GLM은 세 가지 요소로 구성된다:

| 요소 | 설명 | 로지스틱 회귀에서 |
|------|------|------------------|
| **확률 분포** (Random Component) | 반응변수 Y의 분포 | 베르누이 분포 (Bernoulli) |
| **체계적 성분** (Systematic Component) | 선형 예측자 η = Xβ | η = β₀ + β₁x₁ + ... + βₚxₚ |
| **링크 함수** (Link Function) | E(Y)와 η를 연결 | 로짓 함수: logit(p) = log(p/(1-p)) |

일반 선형 회귀(OLS)는 GLM에서 정규분포 + 항등 링크 함수를 사용하는 특수 사례이고, 로지스틱 회귀는 베르누이 분포 + 로짓 링크 함수를 사용하는 특수 사례이다.

**GLM의 의의**: 로지스틱 회귀를 GLM의 프레임워크 안에서 이해하면, 포아송 회귀(Poisson Regression), 프로빗 회귀(Probit Regression) 등 다른 분류/회귀 모형과의 관계를 체계적으로 파악할 수 있다. 논문에서 "왜 로지스틱 회귀를 선택했는가"를 정당화할 때 이 관점이 필요하다.

---

### 1.2 로짓 링크 함수와 시그모이드

#### 로짓 변환의 수학적 구조

확률 p ∈ (0, 1)에 대해:

$$
\text{logit}(p) = \ln\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1 x_1 + \cdots + \beta_p x_p
$$

이 변환의 핵심은 **확률 공간 (0,1)을 실수 전체 (-∞, +∞)로 매핑**하는 것이다.

| 변환 단계 | 범위 | 변환 |
|-----------|------|------|
| 확률 p | (0, 1) | - |
| 오즈 p/(1-p) | (0, +∞) | 확률 → 오즈 |
| 로짓 ln(p/(1-p)) | (-∞, +∞) | 오즈 → 로짓 |

역변환(시그모이드)은:

$$
p = \sigma(\eta) = \frac{1}{1 + e^{-\eta}} = \frac{e^{\eta}}{1 + e^{\eta}}
$$

#### 로짓 vs 프로빗 vs 보완 로그-로그

로지스틱 회귀 외에도 이진 반응변수를 다루는 링크 함수가 있다:

| 링크 함수 | 수식 | 특성 |
|-----------|------|------|
| **로짓 (Logit)** | ln(p/(1-p)) | 대칭, 오즈비 해석 용이 |
| **프로빗 (Probit)** | Φ⁻¹(p) (표준정규 역CDF) | 대칭, 잠재변수 해석 |
| **보완 로그-로그 (cloglog)** | ln(-ln(1-p)) | 비대칭, 희귀 사건에 적합 |

**논문 Tip**: 의학/역학 분야에서는 오즈비 해석이 관행이므로 로짓을 쓰고, 계량경제학에서는 프로빗도 자주 사용한다. 희귀 사건(rare event)에서는 보완 로그-로그를 고려할 수 있다.

### 1.3 최대우도추정 (Maximum Likelihood Estimation, MLE)

기본 강의에서는 "로그 손실을 최소화한다"고만 설명했다. 여기서는 **왜 로그 손실이 나오는지**를 MLE로부터 유도한다.

#### 우도함수 (Likelihood Function)

n개의 독립 관측치 (xᵢ, yᵢ)에 대해, yᵢ ~ Bernoulli(pᵢ)이면:

$$
L(\boldsymbol{\beta}) = \prod_{i=1}^{n} p_i^{y_i}(1-p_i)^{1-y_i}
$$

여기서 pᵢ = σ(xᵢᵀβ)이다.

#### 로그우도함수 (Log-Likelihood)

계산의 편의와 수치적 안정성을 위해 로그를 취한다:

$$
\ell(\boldsymbol{\beta}) = \sum_{i=1}^{n} \left[ y_i \ln(p_i) + (1-y_i)\ln(1-p_i) \right]
$$

**로그 손실(Log Loss)은 음의 로그우도를 n으로 나눈 것**이다:

$$
\text{Log Loss} = -\frac{1}{n}\ell(\boldsymbol{\beta})
$$

따라서 **로그 손실을 최소화하는 것 = 로그우도를 최대화하는 것 = MLE**이다.

#### 그래디언트 (Gradient)

로그우도를 β에 대해 편미분하면:

$$
\frac{\partial \ell}{\partial \beta_j} = \sum_{i=1}^{n} (y_i - p_i) x_{ij}
$$

벡터 형태로:

$$
\nabla_{\boldsymbol{\beta}} \ell = \mathbf{X}^T(\mathbf{y} - \mathbf{p})
$$

이 그래디언트가 0이 되는 β를 찾으면 MLE를 얻는다. 하지만 **닫힌 형태의 해(closed-form solution)가 존재하지 않으므로** 반복적 수치 최적화가 필요하다.

#### 헤시안 (Hessian)

2차 도함수(헤시안)는:

$$
\mathbf{H} = \frac{\partial^2 \ell}{\partial \boldsymbol{\beta} \partial \boldsymbol{\beta}^T} = -\mathbf{X}^T \mathbf{W} \mathbf{X}
$$

여기서 W = diag(pᵢ(1-pᵢ))는 대각 가중치 행렬이다.

헤시안이 항상 **음의 준정부호(negative semi-definite)** 이므로, 로그우도는 **오목(concave)** 함수이다. 즉, **지역 최대값 = 전역 최대값**이므로 경사상승법(또는 경사하강법으로 음의 로그우도 최소화)이 반드시 수렴한다.

**논문 Tip**: "로지스틱 회귀의 비용 함수는 볼록(convex)이므로 전역 최적해가 보장된다"는 점은 신경망 등 비볼록 모델과의 중요한 차별점이다.

### 1.4 최적화 알고리즘

#### 경사하강법 (Gradient Descent)

$$
\boldsymbol{\beta}^{(t+1)} = \boldsymbol{\beta}^{(t)} + \alpha \cdot \mathbf{X}^T(\mathbf{y} - \mathbf{p}^{(t)})
$$

- α: 학습률 (learning rate)
- 수렴 속도가 느릴 수 있음

#### 뉴턴-랩슨법 (Newton-Raphson) / IRLS

$$
\boldsymbol{\beta}^{(t+1)} = \boldsymbol{\beta}^{(t)} - \mathbf{H}^{-1} \nabla \ell
$$

이를 정리하면 **반복 재가중 최소제곱법(IRLS: Iteratively Reweighted Least Squares)** 이 된다:

$$
\boldsymbol{\beta}^{(t+1)} = (\mathbf{X}^T \mathbf{W}^{(t)} \mathbf{X})^{-1} \mathbf{X}^T \mathbf{W}^{(t)} \mathbf{z}^{(t)}
$$

여기서 z⁽ᵗ⁾ = Xβ⁽ᵗ⁾ + W⁻¹(y - p⁽ᵗ⁾)는 조정된 반응변수(adjusted response)이다.

| 알고리즘 | sklearn solver | 특성 |
|---------|---------------|------|
| 경사하강법 변형 | `'sgd'` (SGDClassifier) | 대규모 데이터에 적합 |
| L-BFGS | `'lbfgs'` (기본값) | 준뉴턴법, 중규모 데이터에 효율적 |
| IRLS/뉴턴 | `'newton-cg'`, `'newton-cholesky'` | 수렴 빠름, 소규모 데이터 |
| SAG/SAGA | `'sag'`, `'saga'` | 확률적 평균 경사, L1 지원(saga) |
| liblinear | `'liblinear'` | 소규모, L1/L2 지원 |

```python
from sklearn.linear_model import LogisticRegression

# solver 비교
solvers = ['lbfgs', 'newton-cg', 'liblinear', 'sag', 'saga']
for s in solvers:
    model = LogisticRegression(solver=s, max_iter=1000, random_state=42)
    model.fit(X_train, y_train)
    print(f"solver={s:15s} -> accuracy={model.score(X_test, y_test):.4f}")
```

## 2. 계수의 통계적 해석

### 2.1 오즈비(Odds Ratio)의 심층 이해

기본 강의에서 "계수의 부호가 양이면 양성 확률 증가"라고 배웠다. 중급 수준에서는 **정확한 양적 해석**을 한다.

#### 오즈비의 정확한 의미

피처 xⱼ가 1단위 증가할 때 (다른 변수 고정):

$$
\text{OR}_j = \frac{\text{Odds}(x_j + 1)}{\text{Odds}(x_j)} = e^{\beta_j}
$$

**해석 예시** (타이타닉 데이터):

| 피처 | 계수 (β) | OR = e^β | 해석 |
|------|---------|----------|------|
| Pclass | -1.182 | 0.307 | 등급이 1단위 증가하면(1등→2등) 생존 오즈가 **69.3% 감소** |
| Age | -0.040 | 0.961 | 나이가 1세 증가하면 생존 오즈가 **3.9% 감소** |
| Sex_male | -2.569 | 0.077 | 남성의 생존 오즈는 여성의 **7.7%** (또는 92.3% 낮음) |

#### 연속형 변수에서의 주의점

Age의 OR = 0.961은 "1세 증가" 기준이다. **10세 증가** 단위로 보고하려면:

$$
\text{OR}_{10\text{세}} = e^{10 \times \beta_{\text{Age}}} = e^{10 \times (-0.040)} = 0.670
$$

즉, 10세 증가 시 생존 오즈가 33.0% 감소한다. 논문에서는 해석이 의미 있는 단위를 선택하는 것이 중요하다.

```python
import numpy as np
import pandas as pd

# 오즈비 계산 및 다양한 단위
coefs = pd.Series(model.coef_[0], index=X.columns)

# 1단위 오즈비
or_1unit = np.exp(coefs)

# Age는 10세 단위, 나머지는 1단위
or_custom = or_1unit.copy()
or_custom['Age'] = np.exp(coefs['Age'] * 10)

print("=== 오즈비 (1단위) ===")
print(or_1unit.round(4))
print("\n=== 오즈비 (커스텀 단위) ===")
print(or_custom.round(4))
```

### 2.2 Wald 검정과 신뢰구간

MLE의 점근적 성질에 의해, β̂ 은 대표본에서 근사적으로 정규분포를 따른다:

$$
\hat{\beta}_j \sim N\left(\beta_j, \left[(\mathbf{X}^T \mathbf{W} \mathbf{X})^{-1}\right]_{jj}\right)
$$

#### Wald 검정 (개별 계수의 유의성)

$$
z_j = \frac{\hat{\beta}_j}{\text{SE}(\hat{\beta}_j)}, \quad z_j \sim N(0,1) \text{ under } H_0: \beta_j = 0
$$

| 결과 | 해석 |
|------|------|
| p-value < 0.05 | 해당 피처가 통계적으로 유의 |
| p-value >= 0.05 | 해당 피처의 유의성 부족 (제거 고려) |

#### 오즈비의 95% 신뢰구간

$$
\text{CI}_{95\%}(\text{OR}_j) = \exp\left(\hat{\beta}_j \pm 1.96 \times \text{SE}(\hat{\beta}_j)\right)
$$

**논문 Tip**: 논문에서는 반드시 오즈비와 함께 **95% 신뢰구간**을 보고해야 한다. 신뢰구간이 1을 포함하면 해당 피처는 통계적으로 유의하지 않다.

#### statsmodels를 활용한 상세 통계 결과

```python
import statsmodels.api as sm

# statsmodels는 상수항(intercept)을 명시적으로 추가해야 함
X_with_const = sm.add_constant(X_train)

# 로지스틱 회귀 (GLM with Binomial family)
logit_model = sm.Logit(y_train, X_with_const)
result = logit_model.fit()

# 상세 요약 출력
print(result.summary())

# 오즈비와 95% 신뢰구간
params = result.params
conf = result.conf_int()
conf.columns = ['2.5%', '97.5%']
conf['OR'] = np.exp(params)
conf['OR_2.5%'] = np.exp(conf['2.5%'])
conf['OR_97.5%'] = np.exp(conf['97.5%'])
conf['p-value'] = result.pvalues
print("\n=== 오즈비 및 95% 신뢰구간 ===")
print(conf[['OR', 'OR_2.5%', 'OR_97.5%', 'p-value']].round(4))
```

**statsmodels summary() 출력 해석**:

| 항목 | 의미 |
|------|------|
| **coef** | 회귀 계수 (β̂) |
| **std err** | 표준오차 SE(β̂) |
| **z** | Wald z-통계량 |
| **P>\|z\|** | p-value (양측 검정) |
| **[0.025, 0.975]** | β̂의 95% 신뢰구간 |
| **Log-Likelihood** | 로그우도값 |
| **Pseudo R²** | McFadden의 유사 R² |

### 2.3 우도비 검정 (Likelihood Ratio Test, LRT)

Wald 검정은 개별 계수를 검정하지만, **모형 전체의 유의성**이나 **중첩 모형(nested models)의 비교**에는 우도비 검정이 더 정확하다.

$$
\Lambda = -2(\ell_{\text{reduced}} - \ell_{\text{full}}) \sim \chi^2_{df}
$$

여기서 df = 풀 모형의 파라미터 수 - 축소 모형의 파라미터 수.

```python
from scipy import stats

# 풀 모형 (모든 변수)
full_model = sm.Logit(y_train, sm.add_constant(X_train)).fit(disp=0)

# 축소 모형 (Age 제거)
X_reduced = X_train.drop('Age', axis=1)
reduced_model = sm.Logit(y_train, sm.add_constant(X_reduced)).fit(disp=0)

# 우도비 검정
LR_stat = -2 * (reduced_model.llf - full_model.llf)
df = 1  # 제거한 변수 수
p_value = stats.chi2.sf(LR_stat, df)

print(f"LR 통계량: {LR_stat:.4f}")
print(f"자유도: {df}")
print(f"p-value: {p_value:.4f}")
```

**논문 Tip**: 변수 선택의 정당화에 우도비 검정을 사용하면 리뷰어를 설득하기 쉽다. "Age를 제거해도 모형의 적합도가 통계적으로 유의하게 나빠지지 않았다 (LRT, χ² = 2.31, df = 1, p = 0.128)"와 같이 보고한다.

### 2.4 다중공선성과 VIF (Variance Inflation Factor)

다중공선성은 독립변수들 사이에 높은 상관관계가 존재할 때 발생하며, 계수 추정의 **분산을 증가**시킨다.

$$
\text{VIF}_j = \frac{1}{1 - R_j^2}
$$

여기서 R²ⱼ는 xⱼ를 나머지 독립변수로 회귀했을 때의 결정계수이다.

| VIF 값 | 해석 |
|--------|------|
| 1 | 다중공선성 없음 |
| 1-5 | 약한 다중공선성 (보통 허용) |
| 5-10 | 중간 다중공선성 (주의 필요) |
| > 10 | 심각한 다중공선성 (조치 필요) |

```python
from statsmodels.stats.outliers_influence import variance_inflation_factor

# VIF 계산
X_vif = sm.add_constant(X_train)
vif_data = pd.DataFrame()
vif_data['Feature'] = X_vif.columns
```

## 3. 모델 평가의 심화

### 3.1 혼동 행렬과 파생 지표

기본 강의에서는 정확도(Accuracy)만 다뤘다. 연구에서는 더 정교한 지표가 필요하다.

#### 혼동 행렬 (Confusion Matrix)

|  | 예측: 0 (음성) | 예측: 1 (양성) |
|--|:---:|:---:|
| **실제: 0** | TN (True Negative) | FP (False Positive) - Type I 오류 |
| **실제: 1** | FN (False Negative) - Type II 오류 | TP (True Positive) |

#### 주요 평가 지표

| 지표 | 수식 | 의미 | 언제 중요한가 |
|------|------|------|-------------|
| **정확도 (Accuracy)** | (TP+TN)/(TP+TN+FP+FN) | 전체 정답 비율 | 클래스 균형 시 |
| **정밀도 (Precision)** | TP/(TP+FP) | 양성 예측의 정확성 | FP 비용이 클 때 |
| **재현율 (Recall/Sensitivity)** | TP/(TP+FN) | 실제 양성 포착률 | FN 비용이 클 때 |
| **특이도 (Specificity)** | TN/(TN+FP) | 실제 음성 포착률 | TN이 중요할 때 |
| **F1 Score** | 2×Precision×Recall/(Precision+Recall) | 정밀도-재현율 조화평균 | 불균형 데이터 시 |

```python
from sklearn.metrics import (
    confusion_matrix, classification_report,
    precision_score, recall_score, f1_score
)

# 혼동 행렬
cm = confusion_matrix(y_test, pred)
print("=== 혼동 행렬 ===")
print(cm)

# 상세 보고서
print("\n=== Classification Report ===")
print(classification_report(y_test, pred, target_names=['사망(0)', '생존(1)']))
```

### 3.2 ROC 곡선과 AUC

**ROC(Receiver Operating Characteristic)** 곡선은 임계값을 0에서 1까지 변화시키면서 **TPR(재현율) vs FPR(1-특이도)** 를 그린 곡선이다. **AUC(Area Under Curve)** 는 이 곡선 아래의 면적으로, 모델의 판별력(discriminative ability)을 종합적으로 평가한다.

| AUC 값 | 판별력 |
|--------|--------|
| 0.5 | 무작위 추측 수준 (판별력 없음) |
| 0.6-0.7 | 약한 판별력 |
| 0.7-0.8 | 적절한 판별력 |
| 0.8-0.9 | 좋은 판별력 |
| 0.9-1.0 | 우수한 판별력 |

#### AUC의 확률적 해석

AUC는 "양성 샘플의 예측 확률이 음성 샘플의 예측 확률보다 높을 확률"과 같다:

$$
\text{AUC} = P(\hat{p}_{\text{positive}} > \hat{p}_{\text{negative}})
$$

```python
from sklearn.metrics import roc_curve, roc_auc_score, auc
import matplotlib.pyplot as plt

# ROC 곡선 데이터
prob = model.predict_proba(X_test)[:, 1]
fpr, tpr, thresholds = roc_curve(y_test, prob)
roc_auc = auc(fpr, tpr)

# 시각화
plt.figure(figsize=(8, 6))
plt.plot(fpr, tpr, 'b-', linewidth=2, label=f'ROC curve (AUC = {roc_auc:.3f})')
plt.plot([0, 1], [0, 1], 'r--', linewidth=1, label='Random classifier (AUC = 0.5)')
plt.xlabel('False Positive Rate (1 - Specificity)', fontsize=12)
plt.ylabel('True Positive Rate (Sensitivity)', fontsize=12)
plt.title('ROC Curve', fontsize=14)
plt.legend(fontsize=11)
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()

# 최적 임계값 찾기 (Youden's J statistic)
J = tpr - fpr
optimal_idx = np.argmax(J)
optimal_threshold = thresholds[optimal_idx]
print(f"최적 임계값 (Youden's J): {optimal_threshold:.4f}")
print(f"  TPR = {tpr[optimal_idx]:.4f}, FPR = {fpr[optimal_idx]:.4f}")
```

**논문 Tip**: 정확도 대신 AUC를 주요 성능 지표로 사용하는 것이 연구에서 표준이다. AUC는 임계값에 독립적이며, 불균형 데이터에서도 안정적이다.

### 3.5 Hosmer-Lemeshow 적합도 검정

모델이 데이터를 잘 설명하는지를 통계적으로 검정한다. 예측 확률을 10개 그룹(decile)으로 나누고, 각 그룹의 관찰 빈도와 기대 빈도를 비교한다.

$$
\hat{C} = \sum_{g=1}^{G} \frac{(O_g - E_g)^2}{E_g(1 - E_g/n_g)} \sim \chi^2_{G-2}
$$

- H₀: 모형이 데이터에 적합하다
- p-value > 0.05이면 적합도가 양호

```python
from scipy import stats

def hosmer_lemeshow_test(y_true, y_prob, n_groups=10):
    """Hosmer-Lemeshow 적합도 검정"""
    data = pd.DataFrame({'y': y_true, 'prob': y_prob})
    data['group'] = pd.qcut(data['prob'], n_groups, duplicates='drop')
    
    observed = data.groupby('group')['y'].sum()
    expected = data.groupby('group')['prob'].sum()
    n_per_group = data.groupby('group')['y'].count()
    
    hl_stat = ((observed - expected)**2 / (expected * (1 - expected / n_per_group))).sum()
    df = len(observed) - 2
    p_value = stats.chi2.sf(hl_stat, df)
    
    return hl_stat, df, p_value

hl_stat, df, p_val = hosmer_lemeshow_test(y_test.values, prob)
print(f"Hosmer-Lemeshow 검정: χ² = {hl_stat:.4f}, df = {df}, p = {p_val:.4f}")
if p_val > 0.05:
    print("→ 모형의 적합도가 양호하다 (귀무가설 기각 실패)")
else:
    print("→ 모형의 적합도가 부족하다 (귀무가설 기각)")
```

## 4. 정규화와 변수 선택

### 4.1 L2 정규화 (Ridge)

기본 로지스틱 회귀의 비용 함수에 L2 페널티를 추가한다:

$$
J(\boldsymbol{\beta}) = -\ell(\boldsymbol{\beta}) + \frac{\lambda}{2}\|\boldsymbol{\beta}\|_2^2 = -\ell(\boldsymbol{\beta}) + \frac{\lambda}{2}\sum_{j=1}^{p}\beta_j^2
$$

- 계수를 0에 가깝게 **축소(shrink)** 하지만, 정확히 0으로 만들지는 않음
- 다중공선성이 있을 때 계수 추정을 안정화
- sklearn에서 `penalty='l2'` (기본값)

### 4.2 L1 정규화 (Lasso)

$$
J(\boldsymbol{\beta}) = -\ell(\boldsymbol{\beta}) + \lambda\|\boldsymbol{\beta}\|_1 = -\ell(\boldsymbol{\beta}) + \lambda\sum_{j=1}^{p}|\beta_j|
$$

- 일부 계수를 **정확히 0으로** 만들어 자동 변수 선택 효과
- 고차원 데이터(p >> n)에서 유용
- sklearn에서 `penalty='l1'`, `solver='saga'` 또는 `'liblinear'`

### 4.3 Elastic Net

L1과 L2의 혼합:

$$
J(\boldsymbol{\beta}) = -\ell(\boldsymbol{\beta}) + \lambda\left[\rho\|\boldsymbol{\beta}\|_1 + \frac{1-\rho}{2}\|\boldsymbol{\beta}\|_2^2\right]
$$

- ρ = l1_ratio: L1과 L2의 비율 (0 = 순수 L2, 1 = 순수 L1)
- 상관된 피처 그룹에서 L1보다 안정적
- sklearn에서 `penalty='elasticnet'`, `solver='saga'`

### 4.4 정규화 강도(C) 튜닝

sklearn의 `LogisticRegression`에서 `C = 1/λ`이다. **C가 클수록 정규화가 약하고, 작을수록 강하다.**

```python
from sklearn.model_selection import cross_val_score
import numpy as np

# C 값에 따른 교차검증 성능 비교
C_values = [0.001, 0.01, 0.1, 1, 10, 100]
results = []

for C in C_values:
    model_reg = LogisticRegression(C=C, penalty='l2', solver='lbfgs', max_iter=1000)
    scores = cross_val_score(model_reg, X_train, y_train, cv=5, scoring='roc_auc')
    results.append({
        'C': C, 'mean_auc': scores.mean(), 'std_auc': scores.std()
    })
    print(f"C={C:7.3f}: AUC = {scores.mean():.4f} (+/- {scores.std():.4f})")

# 시각화
results_df = pd.DataFrame(results)
plt.figure(figsize=(8, 5))
plt.semilogx(results_df['C'], results_df['mean_auc'], 'bo-', linewidth=2)
plt.fill_between(
    results_df['C'],
    results_df['mean_auc'] - results_df['std_auc'],
    results_df['mean_auc'] + results_df['std_auc'],
    alpha=0.2
)
plt.xlabel('C (Inverse of regularization strength)', fontsize=12)
plt.ylabel('Mean AUC (5-fold CV)', fontsize=12)
plt.title('Regularization Strength vs. Model Performance', fontsize=14)
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()
```

| **의학/역학** | 질병 위험 요인 분석 | "BMI와 2형 당뇨 발병 위험의 관계" |
| **사회과학** | 이항 결과의 요인 분석 | "교육 수준이 투표 참여에 미치는 영향" |
| **마케팅** | 고객 행동 예측 | "고객 이탈 예측 모델링 및 주요 요인 분석" |
| **금융** | 신용 평가 | "대출 부도 예측과 차별 변수 영향 평가" |
| **교육** | 학업 성취 예측 | "중도 탈락 위험 요인 식별" |
| **환경** | 사건 발생 예측 | "산불 발생 확률에 대한 기상 요인 분석" |

### 6.2 논문에서 결과를 보고하는 방법

#### 필수 보고 항목

1. **모델 설명**: 사용한 모델, 정규화 여부, solver, 하이퍼파라미터
2. **변수 기술 통계**: 각 독립변수의 평균, 표준편차, 범위
3. **다중공선성 진단**: VIF 테이블
4. **주요 결과 테이블**: 계수, 오즈비, 95% CI, p-value
5. **모델 적합도**: Pseudo R², Log-Likelihood, AIC/BIC
6. **판별력**: AUC with 95% CI, 교차검증 결과
7. **보정**: Hosmer-Lemeshow 검정 또는 보정 곡선

#### 논문 결과 테이블 형식 예시

```
Table 2. Logistic Regression Results for Titanic Survival Prediction

Variable         Coef (β)    OR      95% CI          p-value
─────────────────────────────────────────────────────────────
Pclass           -1.182      0.307   [0.219, 0.430]  <0.001 ***
Age (per 10y)    -0.399      0.671   [0.523, 0.861]   0.002 **
Sex_male         -2.569      0.077   [0.045, 0.131]  <0.001 ***
FamilySize       -0.156      0.856   [0.741, 0.988]   0.034 *
Embarked_Q       -0.079      0.924   [0.512, 1.668]   0.793
Embarked_S       -0.236      0.790   [0.521, 1.198]   0.267
─────────────────────────────────────────────────────────────
Model fit: Pseudo R² = 0.342, AUC = 0.862 [0.831, 0.893]
Hosmer-Lemeshow: χ²(8) = 7.23, p = 0.512

* p < 0.05, ** p < 0.01, *** p < 0.001
```

#### 결과 서술 예시

> "로지스틱 회귀 분석 결과, 성별(Sex_male: OR = 0.077, 95% CI [0.045, 0.131], p < 0.001)이 생존 여부에 가장 강한 영향을 미치는 것으로 나타났다. 남성의 생존 오즈는 여성에 비해 약 92.3% 낮았다. 객실 등급(Pclass: OR = 0.307, 95% CI [0.219, 0.430], p < 0.001)도 유의한 예측 변수로, 등급이 한 단계 낮아질 때마다 생존 오즈가 69.3% 감소하였다. 모델의 판별력은 AUC = 0.862로 양호하였으며, Hosmer-Lemeshow 검정에서 적합도가 충분한 것으로 확인되었다 (χ²(8) = 7.23, p = 0.512)."

### 6.3 한계와 대안 모델

| 한계 | 설명 | 대안 |
|------|------|------|
| **선형 결정경계** | 피처 공간에서 선형으로만 분류 가능 | SVM(커널), 신경망 |
| **피처 간 상호작용** | 자동으로 상호작용을 학습하지 않음 | 상호작용 항 수동 추가 또는 트리 모델 |
| **대규모 피처** | 고차원에서 L1 없이는 성능 저하 | Elastic Net, 랜덤 포레스트 |
| **비선형 관계** | 로짓-선형 가정 위반 시 성능 저하 | GAM, XGBoost, 신경망 |
| **이상치 민감** | 극단적 관측치에 영향 받음 | 로버스트 회귀, 트리 기반 모델 |

#### 로지스틱 회귀의 장점 (대안 대비)

1. **해석 가능성(Interpretability)**: 오즈비로 직접 해석 가능 (XAI 불필요)
2. **통계적 추론**: p-value, 신뢰구간, 우도비 검정 등 풍부한 추론 도구
3. **확률 보정**: 자연스럽게 잘 보정된 확률 출력
4. **소규모 데이터**: 적은 데이터에서도 안정적 (딥러닝보다 유리)
5. **규제 요건**: 의료, 금융 등에서 설명 가능 모델 요구 시 적합

### 6.4 논문 주제 아이디어

로지스틱 회귀를 중심으로 쓸 수 있는 연구 방향:

#### A. 방법론 연구

1. **"불균형 데이터에서 로지스틱 회귀의 정규화 전략 비교"**
   - SMOTE + L1/L2/Elastic Net 조합의 성능을 체계적으로 비교
   - 의료 데이터(희귀 질환)에 적용

2. **"고차원 로지스틱 회귀에서의 변수 선택: Lasso vs Stability Selection"**
   - 유전체(genomics) 데이터에서 유의한 SNP를 찾는 문제
   - 변수 선택의 안정성을 Stability Selection으로 개선

3. **"Penalized Logistic Regression과 XGBoost의 해석 가능성-성능 트레이드오프"**
   - 다양한 도메인 데이터셋에서 두 모델의 성능과 해석력 비교
   - SHAP을 활용한 XGBoost의 사후 해석과 로지스틱 회귀의 내재적 해석 비교

#### B. 응용 연구

4. **"COVID-19 중증화 예측을 위한 로지스틱 회귀 기반 위험도 스코어링"**
   - 임상 변수로 중증 전이 확률을 예측하는 스코어카드 개발
   - Nomogram으로 시각화하여 임상 현장 활용

5. **"온라인 학습 환경에서 학생 이탈 예측: 로지스틱 회귀와 학습 분석"**
   - LMS 로그 데이터를 피처로 활용
   - 조기 개입(early intervention)을 위한 임계값 최적화

6. **"소셜 미디어 텍스트의 감성 분류: TF-IDF + 로지스틱 회귀 파이프라인"**
   - 대규모 텍스트 데이터에서 L1 정규화의 피처 선택 효과 분석
   - BERT 등 대형 모델과의 성능-비용 비교
