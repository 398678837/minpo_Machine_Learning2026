## 5.1 분류 문제의 본질 - 왜 선형회귀로 분류하면 안 되는지

### 5.1.1 회귀와 분류의 근본적 차이

4장에서 학습한 선형 회귀(Linear Regression)는 **연속적인 수치**를 예측하는 알고리즘이다. 반면 로지스틱 회귀(Logistic Regression)는 이름에 "회귀"가 들어있지만, 실제로는 **분류(Classification)** 문제에 사용된다. 분류 문제란 입력 데이터가 어떤 **범주(category)**에 속하는지를 예측하는 문제이다.

| 구분 | 선형 회귀 (Linear Regression) | 로지스틱 회귀 (Logistic Regression) |
|------|------------------------------|-------------------------------------|
| **문제 유형** | 회귀 (Regression) | 분류 (Classification) |
| **출력값** | 연속적인 수치 ($-\infty$ ~ $+\infty$) | 클래스 확률 (0~1 사이 값) |
| **예시** | 주택 가격 예측, 보험료 예측 | 생존/사망, 스팸/정상, 합격/불합격 |
| **활성화 함수** | 없음 (항등 함수) | 시그모이드 함수 (Sigmoid) |
| **비용 함수** | MSE (평균제곱오차) | Log Loss (로그 손실) |

### 5.1.2 선형 회귀로 분류하면 발생하는 문제

Menard(2002)는 이진 종속 변수에 OLS(일반최소제곱법)를 적용할 때의 네 가지 핵심 문제를 체계적으로 정리하였다.

**문제 1: 확률 범위 위반**

선형 회귀의 출력 $\hat{y} = w^T x + b$의 범위는 $(-\infty, +\infty)$이다. 그러나 확률은 반드시 $[0, 1]$ 범위에 있어야 한다. 선형 회귀를 분류에 사용하면 예측값이 0 미만이거나 1을 초과하는 **비현실적인 확률**이 산출될 수 있다.

**문제 2: 이분산성 (Heteroscedasticity)**

이진 종속 변수의 오차 분산은 $\hat{y}(1-\hat{y})$로, 예측값에 의존한다. 이는 OLS의 등분산성(homoscedasticity) 가정을 위배한다.

**문제 3: 비정규 오차**

잔차가 이산적 값만 취하므로(실제값이 0 또는 1뿐이므로) 정규성 가정이 위배된다.

**문제 4: 비선형 관계**

확률과 설명 변수 사이의 실제 관계는 직선이 아닌 **S자 곡선(S-curve)**이다. 확률이 0이나 1에 가까울수록 변화량이 줄어드는 **포화(saturation)** 현상이 자연스럽다.

### 5.1.3 로지스틱 회귀의 핵심 아이디어

로지스틱 회귀는 이 모든 문제를 **시그모이드 함수**로 해결한다. 선형 결합의 출력을 시그모이드 함수에 통과시켜 $(0, 1)$ 범위의 확률값으로 변환한다.

```
입력(X) --> 선형 결합(z = w^T x + b) --> 시그모이드(sigma) --> 확률(P) --> 클래스(0 또는 1)
```

이 아이디어는 Cox(1958)가 "The Regression Analysis of Binary Sequences"에서 최초로 체계화하였다.

## 5.2 시그모이드와 로짓 - 오즈비, 로짓 변환

### 5.2.1 시그모이드 함수 (Sigmoid Function)

시그모이드 함수는 로지스틱 회귀의 핵심으로, 임의의 실수값을 0과 1 사이의 값으로 변환한다.

$$
\sigma(z) = \frac{1}{1 + e^{-z}}
$$

여기서 $z = w^T x + b$이다.

**시그모이드 함수의 핵심 성질:**

| 입력값 $z$ | $\sigma(z)$ | 해석 |
|:----------:|:-----------:|:----:|
| $z \to +\infty$ | 1에 가까움 | 양성 클래스(1)일 확률이 높음 |
| $z = 0$ | 0.5 | 양성/음성 확률이 동일 (결정 경계) |
| $z \to -\infty$ | 0에 가까움 | 음성 클래스(0)일 확률이 높음 |

**수학적 성질 (미분):**

$$
\sigma'(z) = \sigma(z) \cdot (1 - \sigma(z))
$$

이 미분이 깔끔한 형태를 가지기 때문에 경사하강법에서 그래디언트 계산이 매우 효율적이다.

### 5.2.2 오즈 (Odds)

**오즈(Odds)**란 어떤 사건이 발생할 확률과 발생하지 않을 확률의 **비율**이다.

$$
\text{Odds} = \frac{P(\text{사건 발생})}{P(\text{사건 미발생})} = \frac{p}{1 - p}
$$

**도박에서의 오즈 개념:**
- 경마에서 "3대 1(3:1) 배당"이란 사건이 발생할 확률이 발생하지 않을 확률의 3배라는 뜻이다.
- 생존 확률 $p = 0.75$이면, Odds $= 0.75 / 0.25 = 3$ ("생존할 확률이 사망할 확률의 3배")
- 합격 확률 $p = 0.50$이면, Odds $= 0.50 / 0.50 = 1$ ("합격과 불합격이 동일")
- 발병 확률 $p = 0.20$이면, Odds $= 0.20 / 0.80 = 0.25$

**의료 사례 - 흡연과 폐암:**
- 흡연자의 폐암 Odds = 0.12 / 0.88 = 0.136
- 비흡연자의 폐암 Odds = 0.01 / 0.99 = 0.0101
- Odds Ratio(오즈비) = 0.136 / 0.0101 = 13.5 ("흡연자는 비흡연자보다 폐암 오즈가 13.5배 높다")

### 5.2.3 로짓 변환 (Logit Transformation)

**로짓(Logit)**은 오즈에 자연로그를 취한 값으로, 시그모이드 함수의 **역함수**이다.

$$
\text{logit}(p) = \ln\left(\frac{p}{1 - p}\right) = z = w^T x + b
$$

이 변환의 핵심적 의의는 **범위의 일치**에 있다:

| 변환 단계 | 범위 |
|:---------:|:----:|
| 확률 $p$ | $(0, 1)$ |
| 오즈 $p/(1-p)$ | $(0, +\infty)$ |
| **로짓** $\ln[p/(1-p)]$ | $(-\infty, +\infty)$ |
| 선형 결합 $z = w^T x + b$ | $(-\infty, +\infty)$ |

즉, 로지스틱 회귀는 **"확률을 로짓 변환하면 피처들의 선형 결합과 같다"**는 가정 하에 작동한다. Cox(1958)가 이 로짓 변환을 체계적으로 도입하여 이진 데이터 분석의 기초를 정립하였다.

### 5.2.4 오즈비 (Odds Ratio)

**오즈비(Odds Ratio)**는 특정 피처가 1단위 증가할 때, 오즈가 몇 배 변하는지를 나타낸다.

$$
\text{Odds Ratio} = e^{w_i}
$$

여기서 $w_i$는 해당 피처의 회귀 계수(coefficient)이다.

| 계수 $w_i$의 부호 | 오즈비 | 해석 |
|:-----------------:|:------:|:----:|
| $w_i > 0$ | $> 1$ | 해당 피처 증가 시 양성 확률 **증가** |
| $w_i = 0$ | $= 1$ | 해당 피처는 결과에 영향 없음 |
| $w_i < 0$ | $< 1$ | 해당 피처 증가 시 양성 확률 **감소** |

**타이타닉 예시:**
- `Sex_male`의 계수 = -2.569이면, Odds Ratio = $e^{-2.569} \approx 0.077$으로, 남성은 여성보다 생존 오즈가 약 **92.3% 낮다**.
- `Pclass`의 계수 = -1.182이면, 등급이 높아질수록(3등실 방향) 생존 오즈가 감소한다.

Menard(2002)는 오즈비 외에도 **표준화 로짓 계수**, **한계 효과(marginal effect)** 등 다양한 계수 해석 방법을 비교하였다.

## 5.3 MLE와 크로스엔트로피 - 우도함수, 로그우도, 경사하강법

### 5.3.1 왜 MSE를 쓰면 안 되는가

시그모이드 함수의 비선형성 때문에 MSE를 비용 함수로 사용하면 **비볼록(non-convex)** 함수가 되어 경사하강법으로 전역 최솟값을 찾기 어렵다. 로지스틱 회귀에는 **로그 손실(Log Loss)**을 사용해야 한다.

### 5.3.2 최대우도추정 (MLE)의 원리

$n$개의 독립적인 관측치 $(x_1, y_1), \ldots, (x_n, y_n)$이 있고, 각 $y_i \in \{0, 1\}$일 때, 단일 관측치의 확률은 다음과 같다:

$$
P(y_i | x_i; w) = \hat{y}_i^{y_i} (1 - \hat{y}_i)^{1 - y_i}
$$

여기서 $\hat{y}_i = \sigma(w^T x_i + b)$이다. 이 식은 다음과 같이 작동한다:
- $y_i = 1$이면: $P = \hat{y}_i$ (양성 확률 자체)
- $y_i = 0$이면: $P = 1 - \hat{y}_i$ (음성 확률)

**우도 함수(Likelihood Function)**는 모든 관측치의 결합 확률이다:

$$
L(w) = \prod_{i=1}^{n} \hat{y}_i^{y_i} (1 - \hat{y}_i)^{1 - y_i}
$$

### 5.3.3 로그 우도에서 크로스엔트로피로

곱셈을 덧셈으로 변환하기 위해 로그를 취한다:

$$
\ell(w) = \ln L(w) = \sum_{i=1}^{n} \left[ y_i \ln(\hat{y}_i) + (1 - y_i) \ln(1 - \hat{y}_i) \right]
$$

MLE는 이 로그 우도를 **최대화**하는 파라미터를 찾는다. 최적화 관행상 최소화 문제로 변환하면, 부호를 반전하고 평균을 취하여 **이진 교차 엔트로피(Binary Cross Entropy)** = **로그 손실(Log Loss)**을 얻는다:

$$
J(w) = -\frac{1}{n}\sum_{i=1}^{n}\left[y_i \ln(\hat{y}_i) + (1 - y_i) \ln(1 - \hat{y}_i)\right]
$$

> **핵심 통찰**: MLE에서 우도를 최대화하는 것 = 로그 손실(크로스엔트로피)을 최소화하는 것. 이 동치성은 Cox(1958)의 논문에서 확립되었다.

### 5.3.4 로그 손실의 직관적 이해

| 실제값 ($y$) | 예측 확률 ($\hat{y}$) | 손실 (Loss) | 해석 |
|:------------:|:---------------------:|:-----------:|:----:|
| 1 | 0.9 | $-\ln(0.9) = 0.105$ | 잘 예측함 - 낮은 손실 |
| 1 | 0.1 | $-\ln(0.1) = 2.303$ | 잘못 예측함 - 높은 손실 |
| 0 | 0.1 | $-\ln(0.9) = 0.105$ | 잘 예측함 - 낮은 손실 |
| 0 | 0.9 | $-\ln(0.1) = 2.303$ | 잘못 예측함 - 높은 손실 |

예측이 틀릴수록 손실이 **기하급수적으로** 증가한다. 이를 통해 모델이 확신에 찬 잘못된 예측에 큰 페널티를 부여한다.

### 5.3.5 그래디언트 유도

비용 함수 $J(w)$에 대한 그래디언트를 구하면:

$$
\frac{\partial J}{\partial w_j} = \frac{1}{n} \sum_{i=1}^{n} (\hat{y}_i - y_i) \cdot x_{ij}
$$

$$
\frac{\partial J}{\partial b} = \frac{1}{n} \sum_{i=1}^{n} (\hat{y}_i - y_i)
$$

벡터 형태로 정리하면:

$$
\nabla_w J = \frac{1}{n} X^T (\hat{y} - y), \quad \frac{\partial J}{\partial b} = \frac{1}{n} \mathbf{1}^T (\hat{y} - y)
$$

이 그래디언트는 Cox(1958)의 **스코어 방정식(score equation)**에서 유도된다.

## 5.4 경사하강법 - 배치/미니배치/SGD 비교, 학습률 선택

### 5.4.1 경사하강법의 원리

경사하강법(Gradient Descent)은 비용 함수의 기울기(그래디언트) 반대 방향으로 파라미터를 반복적으로 업데이트하여 최솟값을 찾는 최적화 알고리즘이다.

$$
w \leftarrow w - \alpha \cdot \nabla_w J(w)
$$

$$
b \leftarrow b - \alpha \cdot \frac{\partial J}{\partial b}
$$

여기서 $\alpha$는 **학습률(learning rate)**이다.

> **참고**: Cox(1958)는 원래 **뉴턴-랩슨(Newton-Raphson)** 방법(2차 미분 활용)을 제안하였다. 경사하강법은 2차 미분을 단위 행렬로 근사한 특수한 경우로 볼 수 있다. sklearn의 `LogisticRegression`은 기본적으로 **L-BFGS**(준뉴턴법)를 사용한다.

### 5.4.2 배치/미니배치/SGD 비교

| 방법 | 한 번 업데이트에 사용하는 데이터 | 장점 | 단점 |
|:----:|:-------------------------------:|:----:|:----:|
| **배치 GD** (Batch) | 전체 데이터셋 ($n$개) | 안정적 수렴, 정확한 그래디언트 | 대용량 데이터에서 느림, 메모리 많이 사용 |
| **미니배치 GD** (Mini-batch) | 일부 ($B$개, 보통 32~256) | 배치와 SGD의 절충, GPU 병렬화 효율적 | 배치 크기 $B$ 하이퍼파라미터 추가 |
| **SGD** (Stochastic) | 단일 샘플 (1개) | 매우 빠른 업데이트, 지역 최솟값 탈출 가능 | 노이즈가 크고 수렴이 불안정 |

**실전에서는 미니배치 GD가 가장 많이 사용된다.**

### 5.4.3 학습률 선택의 중요성

학습률 $\alpha$는 경사하강법에서 가장 중요한 하이퍼파라미터이다.

| 학습률 | 현상 |
|:------:|:----:|
| 너무 작음 ($\alpha = 0.001$) | 수렴이 매우 느림, 반복 횟수 증가 |
| 적절함 ($\alpha = 0.01 \sim 0.1$) | 안정적으로 빠르게 수렴 |
| 너무 큼 ($\alpha = 10$) | 발산(divergence), 비용 함수가 증가 |

**학습률 선택 가이드라인:**
1. 로그 스케일로 탐색: $\{0.001, 0.003, 0.01, 0.03, 0.1, 0.3, 1.0\}$
2. 비용 함수 수렴 곡선을 확인하여 적절한 값 결정
3. 학습률 스케줄링: 학습이 진행됨에 따라 학습률을 점진적으로 감소

### 5.4.4 특성 표준화의 필요성

경사하강법의 수렴을 위해 **특성 표준화(Feature Standardization)**가 필수적이다.

$$
x_j^{\text{scaled}} = \frac{x_j - \mu_j}{\sigma_j}
$$

표준화를 하지 않으면 특성 간 스케일 차이로 인해 비용 함수의 등고선이 타원형이 되어 지그재그 경로로 수렴하게 된다.

## 5.5 다중 클래스 - 소프트맥스, OvR, OvO 전략

### 5.5.1 이진 분류에서 다중 분류로

로지스틱 회귀는 기본적으로 **이진 분류(Binary Classification)** 알고리즘이지만, 확장을 통해 **다중 분류(Multi-class Classification)**에도 사용할 수 있다.

| 구분 | 이진 분류 | 다중 분류 |
|:----:|:---------:|:---------:|
| **클래스 수** | 2개 | $K$개 ($K \geq 3$) |
| **활성화 함수** | 시그모이드 (Sigmoid) | 소프트맥스 (Softmax) |
| **출력** | 1개의 확률값 | $K$개의 확률값 (합 = 1) |
| **예시** | 생존/사망, 스팸/정상 | 붓꽃 종 분류, 숫자 인식(0~9) |

### 5.5.2 소프트맥스 함수 (Softmax Function)

$K$개의 클래스가 있을 때, 각 클래스 $k$에 대한 확률:

$$
P(y = k | X) = \frac{e^{z_k}}{\sum_{j=1}^{K} e^{z_j}}
$$

여기서 $z_k = w_k^T x + b_k$는 클래스 $k$에 대한 선형 결합 결과이다.

**소프트맥스의 핵심 성질:**
- 모든 출력이 $(0, 1)$ 범위
- 모든 클래스에 대한 확률의 합 = 1: $\sum_{k=1}^{K} P(y=k|X) = 1$
- $K=2$일 때 시그모이드 함수와 동일

**다중 클래스의 비용 함수 - 범주형 교차 엔트로피(Categorical Cross Entropy):**

$$
J = -\frac{1}{n}\sum_{i=1}^{n}\sum_{k=1}^{K} y_{ik} \ln(\hat{y}_{ik})
$$

여기서 $y_{ik}$는 $i$번째 샘플이 클래스 $k$에 속하면 1, 아니면 0 (원-핫 인코딩)이다.

### 5.5.3 OvR (One-vs-Rest) 전략

각 클래스에 대해 "해당 클래스 vs 나머지 전부"로 이진 분류기를 $K$개 학습한다.

- $K$개의 클래스가 있으면 $K$개의 이진 분류기 학습
- 예측 시, 가장 높은 확률을 출력하는 분류기의 클래스를 선택
- sklearn: `multi_class='ovr'`

### 5.5.4 OvO (One-vs-One) 전략

모든 클래스 쌍에 대해 이진 분류기를 학습한다.

- $K$개의 클래스가 있으면 $\binom{K}{2} = K(K-1)/2$개의 이진 분류기 학습
- 예측 시, 투표(voting)로 최종 클래스 결정
- 각 분류기가 더 적은 데이터를 사용하므로 개별 학습은 빠르지만, 분류기 수가 많아짐

### 5.5.5 sklearn에서의 다중 분류

```python
from sklearn.linear_model import LogisticRegression

# Multinomial (소프트맥스) - 모든 클래스를 동시에 고려
model_softmax = LogisticRegression(multi_class='multinomial', solver='lbfgs', max_iter=200)

# OvR - 각 클래스에 대해 개별 이진 분류기 학습
model_ovr = LogisticRegression(multi_class='ovr', solver='lbfgs', max_iter=200)
```

## 5.6 판별 vs 생성 모델 - Ng&Jordan(2002) 기반 비교

### 5.6.1 두 가지 근본적 접근법

Ng & Jordan(2002)의 논문 "On Discriminative vs. Generative classifiers"는 기계학습의 두 가지 근본적 접근법을 비교한 획기적 연구이다.

**판별 모델 (Discriminative Model):**
- **학습 대상**: $P(y|x)$ (조건부 분포)를 **직접** 학습
- **대표 모델**: 로지스틱 회귀, SVM, 신경망
- 결정 경계(decision boundary)를 직접 학습하므로 점근적으로 더 정확

**생성 모델 (Generative Model):**
- **학습 대상**: $P(x|y)$와 $P(y)$를 학습한 후 베이즈 정리로 $P(y|x)$ 유도
- **대표 모델**: 나이브 베이즈, 가우시안 혼합 모델(GMM), HMM
- 적은 데이터에서도 효과적, 모델 가정이 맞으면 매우 효율적

### 5.6.2 나이브 베이즈와 로지스틱 회귀의 이론적 관계

Ng & Jordan(2002)의 핵심 발견 중 하나는, 나이브 베이즈 가정(특성 독립성, 지수족 분포) 하에서:

$$
\ln\frac{P(y=1|x)}{P(y=0|x)} = \theta_0 + \sum_j \theta_j x_j
$$

이것이 바로 **로지스틱 회귀의 형태**이다! 즉, 두 모델은 **동일한 모델 패밀리**를 공유하지만, 파라미터를 추정하는 방법이 다르다.

- **나이브 베이즈**: 결합 우도(joint likelihood) $P(x, y)$를 최대화
- **로지스틱 회귀**: 조건부 우도(conditional likelihood) $P(y|x)$를 최대화

### 5.6.3 수렴 속도와 점근적 성능의 트레이드오프

| 특성 | 나이브 베이즈 (생성) | 로지스틱 회귀 (판별) |
|:----:|:--------------------:|:--------------------:|
| 점근적 오차 | 높거나 같음 | **낮거나 같음** |
| 수렴 속도 | $O(\log n)$ (**빠름**) | $O(n)$ (느림) |
| 소표본 성능 | **우수** | 열등 |
| 대표본 성능 | 열등 | **우수** |

### 5.6.4 전환점 (Crossover Point)

Ng & Jordan(2002)의 가장 중요한 실용적 결론:

> **표본이 작을 때는 나이브 베이즈(생성 모델)가 우수하고, 표본이 클 때는 로지스틱 회귀(판별 모델)가 우수하다. 그리고 이 "전환점(crossover point)"이 존재한다.**

이 전환점은 데이터의 특성과 모델 가정의 정확도에 따라 달라진다:
- 나이브 베이즈 가정이 잘 성립하면: 전환점이 늦게 나타남 (더 많은 데이터가 필요해야 로지스틱 회귀가 역전)
- 나이브 베이즈 가정이 위반되면: 전환점이 일찍 나타남

### 5.6.5 편향-분산 트레이드오프 관점

| 관점 | 나이브 베이즈 | 로지스틱 회귀 |
|:----:|:------------:|:------------:|
| **편향 (Bias)** | 높음 (강한 모델 가정) | 낮음 (유연한 추정) |
| **분산 (Variance)** | 낮음 (적은 파라미터) | 높음 (많은 파라미터, 소표본에서 불안정) |
| **소표본** | 낮은 분산이 유리 | 높은 분산이 불리 |
| **대표본** | 높은 편향이 불리 | 낮은 편향이 유리 |

## 5.7 희귀 이벤트 - King&Zeng(2001) 불균형 데이터 편향 보정

### 5.7.1 희귀 이벤트 문제의 정의

King & Zeng(2001)의 논문 "Logistic Regression in Rare Events Data"는 양성 비율이 매우 낮은 **희소 사건(rare events)**에서 표준 로지스틱 회귀의 편향 문제를 분석하였다.

**실전 예시:**

| 분야 | 사례 | 양성 비율 |
|:----:|:----:|:---------:|
| 정치학 | 국제 분쟁 발생 | ~1-3% |
| 금융 | 신용카드 사기 | ~0.1% |
| 의학 | 희귀 질환 진단 | ~1% |
| 제조 | 불량품 탐지 | ~0.5% |
| 정보보안 | 침입 탐지 | ~0.01% |

### 5.7.2 유한표본 편향 (Finite-Sample Bias)

King & Zeng(2001)는 희소 사건에서 MLE의 편향을 이론적으로 분석하였다:

$$
\text{bias}(\hat{\beta}) = (X^T \hat{W} X)^{-1} X^T \hat{W} \xi
$$

핵심 결과: **양성 비율이 낮을수록 편향이 커지며**, 이 편향은 예측 확률의 **과소추정** 방향으로 작용한다. 즉, 희소 사건의 발생 확률을 실제보다 더 낮게 추정하는 경향이 있다.

### 5.7.3 사전 교정 (Prior Correction)

인구 수준의 사건 발생률 $\tau$와 표본에서의 사건 비율 $\bar{y}$를 알 때, 보정된 절편:

$$
\beta_0^{\text{corrected}} = \hat{\beta}_0 - \ln\left[\left(\frac{1-\tau}{\tau}\right)\left(\frac{\bar{y}}{1-\bar{y}}\right)\right]
$$

이 보정은 케이스-통제 설계에서 양성 사례를 과대 표집(oversampling)했을 때의 편향을 교정한다.

### 5.7.4 가중 로지스틱 회귀 (Weighted Logistic Regression)

각 관측치에 역선택확률 가중치(inverse probability weight)를 부여한다:

- 양성 사례($Y=1$)의 가중치: $w_1 = \tau / \bar{y}$
- 음성 사례($Y=0$)의 가중치: $w_0 = (1-\tau) / (1-\bar{y})$

sklearn에서는 `class_weight='balanced'` 옵션으로 구현할 수 있다:

```python
model_weighted = LogisticRegression(class_weight='balanced')
```

이 옵션은 각 클래스의 빈도에 반비례하는 가중치를 자동으로 계산하여 적용한다.

### 5.7.5 실용적 권고사항

King & Zeng(2001)의 논문에서 도출된 실용적 가이드라인:

| 양성 비율 | 권고사항 |
|:---------:|:--------:|
| $> 20\%$ | 표준 로지스틱 회귀 사용 가능 |
| $5\% \sim 20\%$ | 가중 로지스틱 회귀 권고 |
| $< 5\%$ | **반드시** 보정 방법 적용 (가중치 또는 사전 교정) |

추가 권고:
- 평가 지표: accuracy 대신 **AUC, Recall, F1, PR-AUC** 사용
- 임계값: 0.5 대신 양성 비율 또는 비즈니스 요구에 맞게 조정
- 표본 설계: 양성의 2~5배 음성이면 충분

## 5.8 모델 평가 - ROC/AUC, 혼동행렬, Precision/Recall/F1/MCC

### 5.8.1 혼동 행렬 (Confusion Matrix)

분류 모델의 성능을 평가하는 가장 기본적인 도구이다.

|  | 예측: 음성(0) | 예측: 양성(1) |
|:--:|:------------:|:------------:|
| **실제: 음성(0)** | TN (True Negative) | FP (False Positive) - **제1종 오류** |
| **실제: 양성(1)** | FN (False Negative) - **제2종 오류** | TP (True Positive) |

### 5.8.2 주요 평가 지표

**정확도 (Accuracy):**

$$
\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}
$$

불균형 데이터에서는 무의미할 수 있다. 양성이 1%인 데이터에서 모두 음성으로 예측해도 정확도 99%.

**정밀도 (Precision):**

$$
\text{Precision} = \frac{TP}{TP + FP}
$$

양성으로 예측한 것 중 실제 양성의 비율. "예측의 품질"을 측정한다.

**재현율 (Recall / Sensitivity / TPR):**

$$
\text{Recall} = \frac{TP}{TP + FN}
$$

실제 양성 중 양성으로 예측한 비율. "검출력"을 측정한다.

**F1 Score:**

$$
F_1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}
$$

정밀도와 재현율의 **조화평균**. 두 지표의 균형을 측정한다.

**MCC (Matthews Correlation Coefficient):**

$$
\text{MCC} = \frac{TP \cdot TN - FP \cdot FN}{\sqrt{(TP+FP)(TP+FN)(TN+FP)(TN+FN)}}
$$

- 범위: $[-1, +1]$. +1이면 완벽한 예측, 0이면 무작위 수준, -1이면 완전히 반대.
- **불균형 데이터에서 accuracy보다 신뢰할 수 있는 단일 지표**이다.

### 5.8.3 ROC 곡선과 AUC

**ROC 곡선(Receiver Operating Characteristic Curve):**
- x축: 위양성률 (FPR) = $FP / (FP + TN)$
- y축: 진양성률 (TPR = Recall) = $TP / (TP + FN)$
- 다양한 임계값에서의 FPR-TPR 쌍을 연결한 곡선

**AUC(Area Under the ROC Curve):**
- ROC 곡선 아래 면적
- Hosmer & Lemeshow(2000)의 c-통계량(concordance statistic)과 동일

| AUC 범위 | 판별력 |
|:--------:|:------:|
| 0.5 | 무작위 추측 수준 |
| 0.7 ~ 0.8 | 수용 가능한 판별력 |
| 0.8 ~ 0.9 | 우수한 판별력 |
| $> 0.9$ | 뛰어난 판별력 |

### 5.8.4 Precision-Recall 곡선

불균형 데이터에서는 ROC 곡선보다 **PR(Precision-Recall) 곡선**이 더 유용할 수 있다. PR 곡선은 양성 클래스에 초점을 맞추기 때문이다.

- x축: Recall
- y축: Precision
- AP(Average Precision): PR 곡선 아래 면적

### 5.8.5 임계값(Threshold) 조정

기본 임계값 0.5가 항상 최적은 아니다. 문제의 특성에 따라 조정이 필요하다.

| 상황 | 임계값 방향 | 강조 지표 | 이유 |
|:----:|:----------:|:---------:|:----:|
| **질병 진단** | 낮춤 (0.3 등) | Recall | 놓치면 생명 위험 |
| **스팸 필터링** | 높임 (0.7 등) | Precision | 정상 메일 손실 방지 |
| **사기 탐지** | 낮춤 (0.1 등) | Recall | 사기 놓치면 큰 손실 |
| **고객 이탈 예측** | 낮춤 (0.3 등) | Recall | 이탈 방지 실패 비용이 큼 |

### 5.8.6 Hosmer-Lemeshow 적합도 검정

Hosmer & Lemeshow(2000)가 제안한 모델 적합도 검정:

1. 예측 확률을 기준으로 데이터를 $g$개 그룹(보통 10개)으로 분할
2. 각 그룹에서 관측된 사건 수와 기대 사건 수를 비교
3. 검정 통계량: $\hat{C} = \sum_{k=1}^{g} \frac{(O_k - E_k)^2}{E_k(1 - E_k/n_k)}$
4. 귀무가설: "모델이 데이터를 잘 설명한다"
5. p-value가 높으면 모델 적합도가 좋다고 판단

## 5.9 논문 리뷰 - Cox(1958), Hosmer&Lemeshow, Menard 등

### 5.9.1 Cox (1958) - "The Regression Analysis of Binary Sequences"

**핵심 기여:** 로지스틱 회귀의 이론적 토대를 정립한 기념비적 논문이다.

- **로짓 변환의 체계적 도입**: 이진 반응 변수에 대해 $\log(p/(1-p)) = \beta_0 + \beta_1 x_1 + \cdots + \beta_k x_k$로 모델링하는 방법을 정립
- **MLE의 적용**: 로지스틱 회귀 파라미터 추정에 최대우도추정법을 사용하는 방법과 점근적 성질을 체계화
- **프로빗 모델과의 비교**: 프로빗(probit) 모델 대비 로짓 모델의 해석적 장점(오즈비 해석)을 강조
- **응용 분야**: 의학(투약량-반응 관계), 생물학(독성 실험), 사회과학(투표 행동) 등

> Cox의 이 논문을 이해하면, 왜 시그모이드 함수를 사용하는지, 왜 로그 손실을 비용 함수로 사용하는지, 왜 오즈비로 계수를 해석하는지를 근본적으로 이해할 수 있다.

### 5.9.2 Hosmer & Lemeshow (2000) - "Applied Logistic Regression"

**핵심 기여:** 로지스틱 회귀의 실용적 적용에 관한 가장 권위 있는 교과서이다.

- **체계적 모델 구축 프레임워크**: 변수 선택 -> 모델 적합 -> 진단 -> 평가의 순환적 과정을 체계화
- **목적적 변수 선택(Purposeful Selection)**: 7단계 변수 선택 과정 제시
  1. 단변량 분석 (p < 0.25 기준)
  2. 다변량 모델 적합
  3. 유의하지 않은 변수 제거 (p > 0.05)
  4. 교란변수 확인 (20% 이상 계수 변화 시 재포함)
  5. 연속형 변수의 로짓 선형성 확인
  6. 교호작용 추가
  7. 최종 모델 적합도 평가
- **Hosmer-Lemeshow 적합도 검정**: 로지스틱 회귀 모델의 적합도를 평가하는 표준 도구
- **진단 통계량**: 피어슨 잔차, 이탈도 잔차, 레버리지, dfbeta, Cook의 거리

### 5.9.3 King & Zeng (2001) - "Logistic Regression in Rare Events Data"

**핵심 기여:** 희소 사건에서 표준 로지스틱 회귀의 편향 문제를 체계적으로 분석하였다.

- 양성 비율이 낮을수록 MLE가 사건 발생 확률을 과소추정함을 이론적으로 증명
- **사전 교정(Prior Correction)**: 인구 수준 사건 발생률 정보를 활용한 절편 보정
- **가중 로지스틱 회귀**: 케이스-통제 설계에서의 가중 MLE 제안
- Firth(1993)의 편향 축소 방법을 적용한 **보정된 추정량** 제시:

$$
\ell^*(\beta) = \ell(\beta) + \frac{1}{2}\log|I(\beta)|
$$

### 5.9.4 Ng & Jordan (2002) - "On Discriminative vs. Generative classifiers"

**핵심 기여:** 판별 모델과 생성 모델의 근본적 차이를 이론적, 실험적으로 밝힌 논문이다.

- 나이브 베이즈와 로지스틱 회귀가 동일한 모델 패밀리를 공유함을 증명
- 생성 모델의 $O(\log n)$ 수렴 vs 판별 모델의 $O(n)$ 수렴 증명
- 전환점(crossover point)의 존재를 이론적, 실험적으로 확인
- 데이터 양에 따른 모델 선택 전략 제시

### 5.9.5 Menard (2002) - "Applied Logistic Regression Analysis"

**핵심 기여:** OLS와의 체계적 비교를 통한 로지스틱 회귀 입문서이다.

- **다양한 계수 해석법 비교**: 로짓 계수, 오즈비, 표준화 계수, 한계 효과
- **유사 R-squared 정리**: McFadden's, Cox & Snell, Nagelkerke 등 비교

| 지표 | 범위 | 특징 |
|:----:|:----:|:----:|
| McFadden's $R^2$ | $[0, 1)$ | 로그 우도 기반, 최대값이 1에 도달하지 않음 |
| Cox & Snell $R^2$ | $[0, <1)$ | 최대값이 1 미만 |
| Nagelkerke $R^2$ | $[0, 1]$ | Cox & Snell을 정규화, 최대값 1 가능 |
| Count $R^2$ | $[0, 1]$ | 정확 분류 비율, 가장 직관적 |

- **다중공선성 진단**: VIF > 10, Tolerance < 0.1, 조건지수 > 30이면 심각한 다중공선성

```python
# 1. 핵심 함수 구현
# ============================================================

def sigmoid(z):
    """
    시그모이드(로지스틱) 함수

    수식: sigma(z) = 1 / (1 + exp(-z))

    Cox(1958)가 제안한 로짓 변환의 역함수이다.
    임의의 실수값 z를 (0, 1) 범위의 확률값으로 변환한다.

    수치 안정성을 위해 np.clip으로 오버플로를 방지한다.
    """
    # 수치 안정성: z가 너무 크거나 작으면 exp에서 오버플로 발생 가능
    z = np.clip(z, -500, 500)
    return 1.0 / (1.0 + np.exp(-z))


def compute_cost(X, y, weights, bias):
    """
    이진 교차 엔트로피(Binary Cross Entropy) / 로그 손실(Log Loss) 계산

    수식: J(w, b) = -(1/m) * sum[ y*log(y_hat) + (1-y)*log(1-y_hat) ]

    이 비용 함수는 Cox(1958)의 음의 로그우도(negative log-likelihood)와 동일하다.
    MLE(최대우도추정)에서 우도를 최대화하는 것은 이 비용 함수를 최소화하는 것과 같다.
    """
    m = X.shape[0]  # 표본 수

    # 선형 결합: z = X @ w + b
    z = np.dot(X, weights) + bias

    # 시그모이드 함수 적용: 확률 예측
    y_hat = sigmoid(z)

    # 수치 안정성: log(0) 방지
    epsilon = 1e-15
    y_hat = np.clip(y_hat, epsilon, 1 - epsilon)

    # 이진 교차 엔트로피 계산
    cost = -(1.0 / m) * np.sum(
        y * np.log(y_hat) + (1 - y) * np.log(1 - y_hat)
    )

    return cost


def compute_gradients(X, y, weights, bias):
    """
    비용 함수에 대한 그래디언트(기울기) 계산

    수식:
        dJ/dw = (1/m) * X^T @ (y_hat - y)
        dJ/db = (1/m) * sum(y_hat - y)

    이 그래디언트는 Cox(1958)의 스코어 방정식에서 유도된다.
    """
    m = X.shape[0]

    # 예측 확률 계산
    z = np.dot(X, weights) + bias
    y_hat = sigmoid(z)

    # 오차: 예측값 - 실제값
    error = y_hat - y

    # 가중치 그래디언트: (1/m) * X^T @ error
    dw = (1.0 / m) * np.dot(X.T, error)

    # 편향 그래디언트: (1/m) * sum(error)
    db = (1.0 / m) * np.sum(error)

    return dw, db


def logistic_regression_gd(X, y, learning_rate=0.1, n_iterations=1000, verbose=True):
    """
    경사하강법(Gradient Descent)을 사용한 로지스틱 회귀 학습

    Cox(1958)는 뉴턴-랩슨 방법을 제안했으나,
    여기서는 더 직관적인 경사하강법을 사용한다.
    """
    m, n = X.shape

    # 파라미터 초기화 (0으로 시작)
    weights = np.zeros(n)
    bias = 0.0

    # 비용 기록을 위한 리스트
    cost_history = []

    for i in range(n_iterations):
        # 비용 계산
        cost = compute_cost(X, y, weights, bias)
        cost_history.append(cost)

        # 그래디언트 계산
        dw, db = compute_gradients(X, y, weights, bias)

        # 파라미터 업데이트: w = w - lr * dw
        weights -= learning_rate * dw
        bias -= learning_rate * db

        # 100회마다 진행 상황 출력
        if verbose and (i % 100 == 0 or i == n_iterations - 1):
            print(f"  반복 {i:5d}/{n_iterations} | 비용: {cost:.6f}")

    return weights, bias, cost_history


def predict(X, weights, bias, threshold=0.5):
    """
    학습된 파라미터를 사용하여 예측 수행
    """
    z = np.dot(X, weights) + bias
    probabilities = sigmoid(z)
    predictions = (probabilities >= threshold).astype(int)
    return predictions, probabilities
```

## 5.11 실습: 판별 vs 생성 비교

> **참고 논문**: Ng & Jordan (2002) "On Discriminative vs. Generative classifiers"

### 전체 코드

```python
# -*- coding: utf-8 -*-
"""
02_discriminative_vs_generative.py
판별 모델(로지스틱 회귀) vs 생성 모델(나이브 베이즈) 비교

Ng & Jordan (2002) 핵심 결과 재현:
1. 판별 모델(로지스틱 회귀)은 점근적으로 더 낮은 오차를 달성
2. 생성 모델(나이브 베이즈)은 적은 데이터에서 더 빠르게 수렴
3. 전환점(crossover point)이 존재
"""

import numpy as np
import matplotlib.pyplot as plt
from sklearn.linear_model import LogisticRegression
from sklearn.naive_bayes import GaussianNB
from sklearn.model_selection import StratifiedShuffleSplit
from sklearn.metrics import accuracy_score
from sklearn.datasets import make_classification
from sklearn.preprocessing import StandardScaler


# ============================================================
# 1. 핵심 실험 함수
# ============================================================

def compare_models_by_sample_size(X, y, train_fractions, n_repeats=50, test_size=0.3):
    """
    다양한 학습 데이터 크기에서 로지스틱 회귀와 나이브 베이즈 비교
    Ng & Jordan(2002)의 핵심 실험 재현
    """
    n_total = len(y)

    lr_means, lr_stds = [], []
    nb_means, nb_stds = [], []
    actual_sizes = []

    for frac in train_fractions:
        n_train = max(int(n_total * (1 - test_size) * frac), 4)

        lr_scores, nb_scores = [], []

        for seed in range(n_repeats):
            splitter = StratifiedShuffleSplit(n_splits=1, test_size=test_size,
                                              random_state=seed)
            train_idx, test_idx = next(splitter.split(X, y))

            X_test = X[test_idx]
            y_test = y[test_idx]

            rng = np.random.RandomState(seed + 1000)
            if n_train < len(train_idx):
                selected = rng.choice(len(train_idx), n_train, replace=False)
                train_subset = train_idx[selected]
            else:
                train_subset = train_idx

            X_train = X[train_subset]
            y_train = y[train_subset]

            if len(np.unique(y_train)) < 2:
                continue

            scaler = StandardScaler()
            X_train_s = scaler.fit_transform(X_train)
            X_test_s = scaler.transform(X_test)

            # --- 로지스틱 회귀 (판별 모델) ---
            try:
                lr_model = LogisticRegression(max_iter=2000, random_state=42,
                                              solver='lbfgs')
                lr_model.fit(X_train_s, y_train)
                lr_pred = lr_model.predict(X_test_s)
                lr_scores.append(accuracy_score(y_test, lr_pred))
            except Exception:
                pass

            # --- 나이브 베이즈 (생성 모델) ---
            try:
                nb_model = GaussianNB()
                nb_model.fit(X_train_s, y_train)
                nb_pred = nb_model.predict(X_test_s)
                nb_scores.append(accuracy_score(y_test, nb_pred))
            except Exception:
                pass

        if len(lr_scores) > 0 and len(nb_scores) > 0:
            lr_means.append(np.mean(lr_scores))
            lr_stds.append(np.std(lr_scores))
            nb_means.append(np.mean(nb_scores))
            nb_stds.append(np.std(nb_scores))
            actual_sizes.append(n_train)

    return {
        'train_sizes': actual_sizes,
        'lr_means': np.array(lr_means),
        'lr_stds': np.array(lr_stds),
        'nb_means': np.array(nb_means),
        'nb_stds': np.array(nb_stds)
    }


def plot_learning_curves(results, title, ax=None, show_crossover=True):
    """학습 곡선 시각화 - Ng & Jordan(2002) Figure 1 재현"""
    if ax is None:
        fig, ax = plt.subplots(figsize=(10, 6))

    sizes = results['train_sizes']

    ax.plot(sizes, results['lr_means'], 'b-o', linewidth=2, markersize=5,
            label='로지스틱 회귀 (판별)', zorder=3)
    ax.fill_between(sizes,
                     results['lr_means'] - results['lr_stds'],
                     results['lr_means'] + results['lr_stds'],
                     color='blue', alpha=0.15)

    ax.plot(sizes, results['nb_means'], 'r-s', linewidth=2, markersize=5,
            label='나이브 베이즈 (생성)', zorder=3)
    ax.fill_between(sizes,
                     results['nb_means'] - results['nb_stds'],
                     results['nb_means'] + results['nb_stds'],
                     color='red', alpha=0.15)

    if show_crossover:
        diff = results['lr_means'] - results['nb_means']
        for i in range(1, len(diff)):
            if diff[i-1] <= 0 and diff[i] > 0:
                crossover_size = sizes[i-1] + (sizes[i] - sizes[i-1]) * (-diff[i-1]) / (diff[i] - diff[i-1])
                crossover_acc = results['nb_means'][i-1] + (results['nb_means'][i] - results['nb_means'][i-1]) * (-diff[i-1]) / (diff[i] - diff[i-1])
                ax.axvline(x=crossover_size, color='green', linestyle=':', linewidth=2, alpha=0.7)
                ax.annotate(f'전환점\nn={int(crossover_size)}',
                           xy=(crossover_size, crossover_acc),
                           xytext=(crossover_size * 1.3, crossover_acc - 0.03),
                           fontsize=10, color='green',
                           arrowprops=dict(arrowstyle='->', color='green'))
                break

    ax.set_xlabel('학습 데이터 크기', fontsize=12)
    ax.set_ylabel('테스트 정확도', fontsize=12)
    ax.set_title(title, fontsize=14)
    ax.legend(fontsize=11, loc='lower right')
    ax.grid(True, alpha=0.3)
    return ax


# ============================================================
# 2. 세 가지 실험 실행
# ============================================================

train_fractions = [0.01, 0.02, 0.05, 0.08, 0.1, 0.15, 0.2, 0.3, 0.5, 0.7, 1.0]

# 실험 1: 나이브 베이즈 가정이 성립하는 데이터

    n_samples=2000, n_features=20, n_informative=20,
    n_redundant=0, n_clusters_per_class=1, class_sep=1.0, random_state=42
)

# 실험 2: 나이브 베이즈 가정이 위반되는 데이터

    n_samples=2000, n_features=20, n_informative=10,
    n_redundant=10, n_clusters_per_class=2, class_sep=0.8,
    flip_y=0.05, random_state=42
)
results2 = compare_models_by_sample_size(X2, y2, train_fractions, n_repeats=50)

# 실험 3: 고차원 데이터
X3, y3 = make_classification(
    n_samples=2000, n_features=50, n_informative=30,
    n_redundant=10, n_clusters_per_class=1, class_sep=1.0, random_state=42
)
results3 = compare_models_by_sample_size(X3, y3, train_fractions, n_repeats=50)


# ============================================================
# 3. 종합 시각화
# ============================================================

fig, axes = plt.subplots(1, 3, figsize=(20, 6))

plot_learning_curves(results1, "실험 1: 독립적 특성\n(나이브 베이즈 가정 성립)", ax=axes[0])
plot_learning_curves(results2, "실험 2: 상관된 특성\n(나이브 베이즈 가정 위반)", ax=axes[1])
plot_learning_curves(results3, "실험 3: 고차원 데이터\n(50개 특성)", ax=axes[2])

plt.suptitle('Ng & Jordan (2002) 재현: 판별 모델 vs 생성 모델 학습 곡선 비교',
             fontsize=16, y=1.03)
plt.tight_layout()
plt.savefig('discriminative_vs_generative_learning_curves.png', dpi=150, bbox_inches='tight')
plt.show()
```

### 코드 해설

**실험 설계의 핵심**: 학습 데이터 크기를 1%에서 100%까지 점진적으로 증가시키며, 각 크기에서 50회 반복 실험하여 평균과 표준편차를 산출한다. 이는 Ng & Jordan(2002)의 실험 방법론을 재현한 것이다.

**세 가지 실험의 의미**:
- **실험 1** (독립적 특성): 나이브 베이즈 가정이 성립하므로, 소표본에서 나이브 베이즈가 우세하고 전환점이 관찰됨
- **실험 2** (상관된 특성): 나이브 베이즈 가정 위반으로, 로지스틱 회귀가 더 빨리 우세해짐
- **실험 3** (고차원): 특성 수가 많아 소표본에서 로지스틱 회귀가 불안정, 나이브 베이즈의 $O(\log n)$ 수렴 이점이 부각됨

## 5.12 실습: 희귀 이벤트 보정

> **참고 논문**: King & Zeng (2001) "Logistic Regression in Rare Events Data"

### 전체 코드

```python
# -*- coding: utf-8 -*-
"""
03_rare_events_logistic.py
희소 사건 데이터에서의 로지스틱 회귀 문제와 교정 방법

King & Zeng (2001) 핵심 결과 재현:
1. 표준 로지스틱 회귀가 희소 사건의 확률을 과소추정하는 문제 입증
2. 사전 교정(Prior Correction)을 통한 절편 보정
3. 가중 로지스틱 회귀(Weighted Logistic Regression)를 통한 편향 교정
4. ROC 곡선과 보정 곡선(Calibration Plot) 비교
"""

import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import (
    accuracy_score, roc_curve, roc_auc_score,
    precision_recall_curve, average_precision_score,
    classification_report, confusion_matrix
)
from sklearn.preprocessing import StandardScaler
from sklearn.calibration import calibration_curve


# ============================================================
# 1. 불균형 데이터 생성 함수
# ============================================================

def create_imbalanced_data(n_samples=10000, positive_rate=0.02,
                           n_features=10, random_state=42):
    """희소 사건(rare events) 데이터셋 생성"""
    X, y = make_classification(
        n_samples=n_samples, n_features=n_features,
        n_informative=6, n_redundant=2, n_clusters_per_class=1,
        weights=[1 - positive_rate, positive_rate],
        class_sep=1.5, flip_y=0.01, random_state=random_state
    )
    return X, y


# ============================================================
# 2. 사전 교정 (Prior Correction) 구현
# ============================================================

def prior_correction(model, population_rate, sample_rate):
    """
    King & Zeng(2001)의 사전 교정 방법
    수식: beta_0_corrected = beta_0 - ln[(1-tau)/tau * y_bar/(1-y_bar)]
    """
    correction = np.log(
        ((1 - population_rate) / population_rate) *
        (sample_rate / (1 - sample_rate))
    )
    return correction


def predict_with_prior_correction(model, X, population_rate, sample_rate):
    """사전 교정이 적용된 예측 확률 계산"""
    logits = np.dot(X, model.coef_[0]) + model.intercept_[0]
    correction = prior_correction(model, population_rate, sample_rate)
    corrected_logits = logits - correction
    corrected_prob = 1.0 / (1.0 + np.exp(-corrected_logits))
    return corrected_prob


# ============================================================
# 3. 실험 1: 희소 사건에서의 편향 입증
# ============================================================

print("=" * 70)
print("실험 1: 다양한 불균형 비율에서의 로지스틱 회귀 편향 분석")
print("=" * 70)

positive_rates = [0.50, 0.20, 0.10, 0.05, 0.02, 0.01]
n_simulations = 30

print(f"\n{'양성비율':>8s} | {'실제비율':>8s} | {'예측평균':>8s} | {'편향':>8s} | {'과소추정%':>10s}")
print("-" * 60)

bias_results = []

for rate in positive_rates:
    pred_means = []
    for seed in range(n_simulations):
        X, y = create_imbalanced_data(n_samples=5000, positive_rate=rate, random_state=seed)
        X_train, X_test, y_train, y_test = train_test_split(
            X, y, test_size=0.3, random_state=seed, stratify=y
        )
        scaler = StandardScaler()
        X_train_s = scaler.fit_transform(X_train)
        X_test_s = scaler.transform(X_test)

        model = LogisticRegression(max_iter=2000, random_state=42)
        model.fit(X_train_s, y_train)

        prob = model.predict_proba(X_test_s)[:, 1]
        pred_means.append(prob.mean())

    avg_pred_mean = np.mean(pred_means)
    bias = avg_pred_mean - rate
    underestimate_pct = (1 - avg_pred_mean / rate) * 100 if rate > 0 else 0

    bias_results.append({
        'rate': rate, 'predicted': avg_pred_mean,
        'bias': bias, 'underestimate': underestimate_pct
    })

    print(f"{rate:>8.2%} | {rate:>8.4f} | {avg_pred_mean:>8.4f} | "
          f"{bias:>+8.4f} | {underestimate_pct:>9.1f}%")

# 4. 실험 2: 교정 방법 비교
# ============================================================

print("\n" + "=" * 70)
print("실험 2: 교정 방법 비교")
print("=" * 70)

scaler = StandardScaler()

sample_positive_rate = y_train.mean()

# 모델 1: 표준 로지스틱 회귀
model_standard = LogisticRegression(max_iter=2000, random_state=42)

for name, pred, prob in [
    ("표준 LR", pred_standard, prob_standard),

# ROC 곡선
ax1 = axes[0]
for name, prob, color in models_info:
    fpr, tpr, _ = roc_curve(y_test, prob)
    auc = roc_auc_score(y_test, prob)
    ax1.plot(fpr, tpr, color=color, linewidth=2, label=f'{name} (AUC={auc:.4f})')
ax1.plot([0, 1], [0, 1], 'k--', linewidth=1, alpha=0.5, label='무작위 기준선')
ax1.set_xlabel('위양성률 (FPR)', fontsize=12)
ax1.set_ylabel('진양성률 (TPR)', fontsize=12)
ax1.set_title('ROC 곡선 비교', fontsize=14)
ax1.legend(fontsize=10, loc='lower right')
ax1.grid(True, alpha=0.3)

# PR 곡선
ax2 = axes[1]
for name, prob, color in models_info:
    precision, recall, _ = precision_recall_curve(y_test, prob)
    ap = average_precision_score(y_test, prob)
    ax2.plot(recall, precision, color=color, linewidth=2, label=f'{name} (AP={ap:.4f})')
baseline = y_test.mean()
ax2.axhline(y=baseline, color='black', linestyle='--', linewidth=1, alpha=0.5,
            label=f'기준선 (양성비율={baseline:.3f})')
ax2.set_xlabel('재현율 (Recall)', fontsize=12)
ax2.set_ylabel('정밀도 (Precision)', fontsize=12)
ax2.set_title('Precision-Recall 곡선 비교', fontsize=14)
ax2.legend(fontsize=10, loc='upper right')
ax2.grid(True, alpha=0.3)

plt.suptitle(f'희소 사건 데이터 (양성 비율: {POSITIVE_RATE:.0%}) - 모델 비교',
             fontsize=15, y=1.02)
plt.tight_layout()
plt.savefig('rare_events_roc_pr_curves.png', dpi=150, bbox_inches='tight')
plt.show()


# ============================================================
# 6. 임계값 분석
# ============================================================

print("\n" + "=" * 70)
print("임계값(Threshold) 분석")
print("=" * 70)

thresholds = [0.01, 0.02, 0.05, 0.1, 0.2, 0.3, 0.5]

print(f"\n{'임계값':>8s} | {'정확도':>8s} | {'정밀도':>8s} | {'재현율':>8s} | {'F1':>8s}")
print("-" * 55)

for threshold in thresholds:
    pred_t = (prob_standard >= threshold).astype(int)
    cm = confusion_matrix(y_test, pred_t)
    tn, fp, fn, tp = cm.ravel()
    acc = accuracy_score(y_test, pred_t)
    precision = tp / (tp + fp) if (tp + fp) > 0 else 0
    recall = tp / (tp + fn) if (tp + fn) > 0 else 0
    f1 = 2 * precision * recall / (precision + recall) if (precision + recall) > 0 else 0
    print(f"{threshold:>8.2f} | {acc:>8.4f} | {precision:>8.4f} | {recall:>8.4f} | {f1:>8.4f}")

print("\n주의: 양성 비율이 2%이므로, 임계값=0.5에서는 양성을 거의 검출하지 못함")
```

### 코드 해설

**사전 교정(Prior Correction)의 구현**: King & Zeng(2001)의 수식 $\beta_0^{\text{corrected}} = \hat{\beta}_0 - \ln[(1-\tau)/\tau \cdot \bar{y}/(1-\bar{y})]$을 직접 구현하였다. 원래 모델의 로짓 값에서 보정 항을 빼서 보정된 확률을 산출한다.

**가중 로지스틱 회귀**: sklearn의 `class_weight='balanced'` 옵션은 King & Zeng의 가중 추정법에 대응한다. 양성 클래스에 더 높은 가중치를 부여하여 희소 사건의 검출력을 향상시킨다.

**임계값 분석**: 양성 비율이 2%인 데이터에서 기본 임계값 0.5는 양성을 거의 검출하지 못한다. 임계값을 낮추면 재현율이 증가하지만 정밀도가 감소하는 트레이드오프가 존재한다.

```python
# 사기 탐지 모델 예시
from sklearn.linear_model import LogisticRegression

fraud_model = LogisticRegression(

```

### 5.13.2 질병 진단 (Disease Diagnosis)

**문제 특성:**
- 질병의 유병률에 따라 불균형 정도가 다양
- 오진의 비용이 매우 높음 (특히 FN: 환자를 정상으로 판단)
- 해석 가능성이 중요: 의사가 모델의 판단 근거를 이해해야 함

**로지스틱 회귀의 장점:**
- **오즈비 해석**: 각 위험 인자가 질병 발생에 미치는 영향을 정량적으로 설명 가능
- **확률 출력**: "이 환자의 당뇨병 발생 확률은 73%"와 같이 직관적 소통 가능
- **계수 해석**: Cox(1958)가 확립한 오즈비 기반 해석이 의학 연구의 표준

```python
# 질병 진단 모델 예시
model = LogisticRegression()
model.fit(X_train, y_train)

# 오즈비 계산
import numpy as np
import pandas as pd

odds_ratios = np.exp(model.coef_[0])
feature_importance = pd.DataFrame({
    'Feature': feature_names,
    'Coefficient': model.coef_[0],
    'Odds Ratio': odds_ratios
}).sort_values('Odds Ratio', ascending=False)

print(feature_importance)
# 예: BMI의 오즈비가 1.85이면, BMI가 1단위 증가할 때 당뇨 오즈가 85% 증가
```

**임계값**: 질병 진단에서는 **Recall**을 극대화하기 위해 임계값을 낮추는 것이 일반적이다 (예: 0.3). 놓치면 환자의 생명이 위험할 수 있기 때문이다.

### 5.13.3 고객 이탈 예측 (Customer Churn Prediction)

**문제 특성:**
- 일반적으로 이탈률 15~30% (중간 수준의 불균형)
- FN(이탈 고객을 놓침)의 비용 > FP(유지 고객에게 불필요한 마케팅)의 비용
- 비즈니스 가치: 신규 고객 유치 비용이 기존 고객 유지 비용의 5~7배

**로지스틱 회귀의 활용:**

```python
# 고객 이탈 예측 모델
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import confusion_matrix, classification_report

model_churn = LogisticRegression(max_iter=1000, random_state=42)
model_churn.fit(X_train, y_train)

pred = model_churn.predict(X_test)
prob = model_churn.predict_proba(X_test)[:, 1]

# 혼동 행렬 분석
cm = confusion_matrix(y_test, pred)
tn, fp, fn, tp = cm.ravel()

print(f"True Negative  (유지->유지 예측): {tn}명")
print(f"False Positive (유지->이탈 예측): {fp}명 <- 불필요한 마케팅 비용")
print(f"False Negative (이탈->유지 예측): {fn}명 <- 이탈 방지 실패 (가장 위험)")
print(f"True Positive  (이탈->이탈 예측): {tp}명 <- 리텐션 마케팅 대상")
```

**비즈니스 전략:**
- 예측 확률 상위 20% 고객에게 리텐션 마케팅 (쿠폰, 할인 등)
- 이탈 주요 원인(높은 계수를 가진 피처) 분석 -> 서비스 개선
- 임계값을 낮추어(예: 0.3) Recall을 높이는 방향으로 조정

#### 1) 로지스틱 회귀 알고리즘

| 항목 | 내용 |

| **비용 함수** | 로그 손실 (Log Loss / Binary Cross Entropy) |

| **출력** | 클래스 확률 (0~1 사이) |
| **장점** | 구현 간단, 확률 해석 가능, 계수로 피처 영향도 파악, 연산 효율적 |

#### 3) 주요 수식 정리

| 수식 | 의미 |
|:----:|:----:|
| $\sigma(z) = \frac{1}{1+e^{-z}}$ | 시그모이드 함수 |
| $\text{logit}(p) = \ln\frac{p}{1-p}$ | 로짓 (시그모이드 역함수) |
| $J = -\frac{1}{n}\sum[y\ln\hat{y}+(1-y)\ln(1-\hat{y})]$ | 이진 교차 엔트로피 |
| $\nabla_w J = \frac{1}{n}X^T(\hat{y}-y)$ | 그래디언트 |
| $\text{OR} = e^{w_i}$ | 오즈비 |
| $P(y=k|X) = \frac{e^{z_k}}{\sum_j e^{z_j}}$ | 소프트맥스 |

> **힌트**: 전환점(crossover point)의 개념과, 데이터 양에 따른 모델 선택 전략을 설명한다.

**질문 8.** 소프트맥스 함수와 시그모이드 함수의 관계를 수학적으로 설명하시오. $K=2$일 때 소프트맥스가 시그모이드로 환원됨을 보이시오.
